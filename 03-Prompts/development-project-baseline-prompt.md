# 开发项目基线 Prompt

## 目标

帮助 Agent 为新开发项目建立项目级 `AGENTS.md` 或等价规则文件,让 AI 编程助手先理解项目再动代码。

## 适用 Agent

- Codex
- Claude Code
- Hermes Agent
- 其他开发 Agent

## System Prompt

```text
请为当前项目建立开发型 Agent 基线。基线只写长期稳定规则,不要混入单次任务要求。

必须覆盖:
1. 宪法规则:语言、改代码前读上下文、最小改动、验证要求。
2. 第一性原理:真实目标、已知事实、缺失上下文、最小可验证修改。
3. 项目定位:项目名称、业务领域、核心用户、主要模块、当前阶段。
4. 代码组织:后端、前端、公共库、基础设施、测试目录。
5. 技术栈:后端、前端、数据库、缓存、队列、构建工具。
6. 工程约束:接口契约、SQL、权限、缓存、并发、生成代码、外部接口。
7. 默认工作方式:定位模块、梳理链路、识别风险、最小实现、验证。
8. 输出要求:改了什么、为什么改、如何验证、无法验证的原因。

写作要求:
- 规则必须可执行,避免抽象口号。
- 项目事实和单次任务要求分开。
- 不确定的事实标记为待确认,不要编造。
- 只保留团队愿意长期维护的内容。
```
