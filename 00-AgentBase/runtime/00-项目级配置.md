# Runtime 项目级配置(`.agent/` 统一规范)

> **本规范 v2.0 修订(2026-07-10)**:统一项目级目录名为 `.agent/`,AGENTS.md 放在项目根。
>
> 不再让不同 Runtime 各用各的目录名(`.trae/`、`.claude/`、`.hermes/`、`.codex/`)——为了一致性,统一到 `.agent/`,通过软链接或 Runtime 原生配置对接。

## 0. 为什么要统一目录名

### 0.1 旧做法的问题

| Runtime | 原生目录 |
|---------|----------|
| Trae | `.trae/` |
| Claude Code | `.claude/` |
| Hermes Agent | `.hermes/` |
| Codex CLI | `.codex/` |

每次换 Runtime 都要:
- 重新命名目录
- 改 `.gitignore`
- 改部署脚本
- 重新写 AGENTS.md
- 团队成员各自维护不同目录

### 0.2 新规范

```
你的项目/
├── AGENTS.md              ← 项目指令(项目根,所有 Runtime 都能读)
├── .agent/                ← 项目级资产目录(唯一规范)
│   ├── skills/            ← 项目级技能
│   ├── agents/            ← 项目级子代理定义(Claude Code 等)
│   ├── plugins/           ← 项目级插件
│   ├── prompts/           ← 项目级 prompt 模板
│   ├── mcp/               ← 项目级 MCP 配置
│   ├── hooks/             ← 项目级 hooks
│   └── memory/            ← 项目级长期记忆
└── 你的项目文件...
```

**核心原则**:
1. **AGENTS.md 在项目根**:所有 Runtime 都自动读
2. **`.agent/` 在项目根**:统一目录名,跨 Runtime 一致
3. **Runtime 适配层在内部隐藏**:通过符号链接或 Runtime 配置,把 `.agent/` 映射到 Runtime 原生位置
4. **.gitignore 统一**:每个项目都加 `.agent/` 或按需选择

## 1. 目录结构

### 1.1 完整结构

```
你的项目/
├── AGENTS.md                       # 项目指令(项目根,所有 Runtime 读)
├── .agent/
│   ├── README.md                   # .agent/ 目录说明(可选)
│   ├── skills/                     # 项目级技能(平铺存放,不再分 global/projects)
│   │   ├── code-review/
│   │   │   └── SKILL.md
│   │   ├── refactor/
│   │   │   └── SKILL.md
│   │   └── kb-init/
│   │       ├── SKILL.md
│   │       └── references/
│   │
│   ├── agents/                     # 项目级子代理(Claude Code 等)
│   │   ├── code-reviewer.md
│   │   └── test-runner.md
│   │
│   ├── plugins/                    # 项目级插件配置
│   │   └── README.md
│   │
│   ├── prompts/                    # 项目级 prompt 模板
│   │   └── README.md
│   │
│   ├── mcp/                        # 项目级 MCP server 配置
│   │   ├── filesystem.json
│   │   └── github.json
│   │
│   ├── hooks/                      # 项目级 hooks(Claude Code 等)
│   │   └── pre-commit.sh
│   │
│   └── memory/                     # 项目级长期记忆
│       └── project-context.md
└── 你的项目文件...
```

### 1.2 创建脚本

**PowerShell(Windows)**:

```powershell
# 在项目根目录运行
New-Item -ItemType Directory -Path ".\.agent\skills" -Force | Out-Null
New-Item -ItemType Directory -Path ".\.agent\agents" -Force | Out-Null
New-Item -ItemType Directory -Path ".\.agent\plugins" -Force | Out-Null
New-Item -ItemType Directory -Path ".\.agent\prompts" -Force | Out-Null
New-Item -ItemType Directory -Path ".\.agent\mcp" -Force | Out-Null
New-Item -ItemType Directory -Path ".\.agent\hooks" -Force | Out-Null
New-Item -ItemType Directory -Path ".\.agent\memory" -Force | Out-Null
New-Item -ItemType File -Path ".\AGENTS.md" -Force | Out-Null

Write-Host "✅ .agent/ 项目级目录已创建"
```

**Bash(macOS / Linux)**:

```bash
mkdir -p .agent/{skills,agents,plugins,prompts,mcp,hooks,memory}
touch AGENTS.md
echo "✅ .agent/ 项目级目录已创建"
```

### 1.3 `.gitignore` 配置

**默认**(全部不入库):

```gitignore
# .agent/ 项目级配置(派生,不入库)
.agent/
```

**推荐**(选择性入库,便于团队共享):

```gitignore
# .agent/ 中的运行时状态不入库
.agent/cache/
.agent/memory/.session/
.agent/mcp/*-local.json

# 这些需要团队共享
# .agent/skills/         ← 入库
# .agent/agents/         ← 入库
# .agent/prompts/        ← 入库
# .agent/hooks/          ← 入库
# AGENTS.md              ← 入库(项目根)
```

**判断标准**:
- **可重建的**(从资产层/外部资源生成)→ 不入库
- **团队约定的**(项目说明、规范、技能)→ 入库
- **包含密钥的**(MCP 配置、个人记忆)→ 不入库

## 2. AGENTS.md(项目根)

### 2.1 作用

`AGENTS.md` 在项目根,**所有 Runtime 都自动读取**(Claude Code 也认 `AGENTS.md`,不只认 `CLAUDE.md`)。

它告诉 Agent:
- 项目是什么
- 用什么技术栈
- 有哪些约定
- 不要做什么

### 2.2 模板(强制 7 节)

```markdown
# 项目名 — 一句话说明

> 项目根的 AGENTS.md。所有 Runtime(Claude Code / Codex / Hermes)启动时都会读。
>
> **告诉 Agent 读哪里**:本项目的技能、插件、Prompt、MCP、子代理在 `.agent/` 下,不要去 .claude/ .codex/ .hermes/ 等目录找。

---

## 1. 项目是什么
(一段话讲清楚项目目的)

## 2. 技术栈
- 语言:
- 框架:
- 测试:
- 部署:

## 3. 关键约定
- (团队/项目的开发约定,每条一行)

## 4. 常用命令
- 开发:`...`
- 测试:`...`
- 构建:`...`
- 部署:`...`

## 5. 边界(不要做的事)
- 不要修改:
- 不要提交:
- 不要调用:
- 需要确认才能做:

## 6. 项目级资产(都在 `.agent/` 下)
- 技能:`.agent/skills/<skill-name>/SKILL.md`
- 子代理:`.agent/agents/<name>.md`(Claude Code / Codex)
- Prompt:`.agent/prompts/<name>.md`
- MCP:`.agent/mcp/<server-name>.json`
- Hooks:`.agent/hooks/<name>.sh`
- 长期记忆:`.agent/memory/<name>.md`

## 7. 资产来源(只读,告诉 Agent 别处)
- 全局技能:`<path-to-knowledge-base>/01-Skills/`
- 知识库:`<path-to-knowledge-base>/00-AgentBase/`
- 运行时知识:`<path-to-knowledge-base>/00-AgentBase/runtime/`
```

### 2.3 模板字段说明

| 字段 | 必填 | 作用 |
|------|------|------|
| 1. 项目是什么 | ✅ | 让 Agent 一句话理解项目 |
| 2. 技术栈 | ✅ | 影响 Agent 的建议和工具选择 |
| 3. 关键约定 | ✅ | 团队规范,Agent 遵守 |
| 4. 常用命令 | ✅ | Agent 知道怎么跑测试/构建 |
| 5. 边界 | ✅ | 防止 Agent 做危险操作 |
| 6. 项目级资产 | ✅ | **核心**:告诉 Agent 去 `.agent/` 找资产 |
| 7. 资产来源 | ✅ | 指向知识库,Agent 可查阅 |

**第 6 节是关键的"导航"**——很多 Agent 默认会去 `.claude/` `.codex/` 找资产,必须明确告诉它去 `.agent/`。

## 3. 各 Runtime 如何对接 `.agent/`

不同 Runtime 仍然有自己的"原生目录"。三种对接方式:

### 3.1 软链接(推荐,开发期)

**思路**:把 `.agent/` 的子目录软链接到 Runtime 原生位置,两边看到同一份内容。

**Claude Code**(把 `.agent/skills/` 链接到 `.claude/skills/`):

```powershell
# Windows PowerShell(需要开发者模式)
New-Item -ItemType Junction -Path ".\.claude\skills" -Target "..\.agent\skills"
New-Item -ItemType Junction -Path ".\.claude\agents" -Target "..\.agent\agents"
```

```bash
# macOS/Linux
ln -s ../.agent/skills .claude/skills
ln -s ../.agent/agents .claude/agents
```

**Codex CLI**:

```powershell
New-Item -ItemType Junction -Path ".\.codex\skills" -Target "..\.agent\skills"
```

**Hermes Agent**:见 [Hermes 外部目录配置](./02-Hermes-Agent/05-技能系统.md#external_dirs)

**Trae**:

```powershell
New-Item -ItemType Junction -Path ".\.trae\skills" -Target "..\.agent\skills"
New-Item -ItemType Junction -Path ".\.trae\AGENTS.md" -Target "..\AGENTS.md"
```

### 3.2 复制(发布/部署期)

**思路**:把 `.agent/` 内容复制到 Runtime 原生位置。稳定可控,但改了要重新复制。

```powershell
# Claude Code
Copy-Item -Recurse -Force .\.agent\skills\* .\.claude\skills\
Copy-Item -Recurse -Force .\.agent\agents\* .\.claude\agents\

# Codex CLI
Copy-Item -Recurse -Force .\.agent\skills\* .\.codex\skills\

# Trae
Copy-Item -Recurse -Force .\.agent\skills\* .\.trae\skills\
Copy-Item -Force .\.agent\AGENTS.md .\.trae\AGENTS.md
```

### 3.3 Runtime 配置(指路径,不动目录)

部分 Runtime 支持配置文件指定资产路径,这种情况下**不需要软链/复制**。

**Codex CLI**:`~/.codex/config.toml`:

```toml
[skills]
external_dirs = ["./.agent/skills"]
```

**Hermes Agent**:见 [Hermes 工具与工具集](./02-Hermes-Agent/04-工具与工具集.md)

**Claude Code**:Claude Code 不支持外部目录配置,只能用软链/复制。

### 3.4 MCP 配置对接

**统一在 `.agent/mcp/` 下放各 Runtime 的 MCP 配置**:

```
.agent/mcp/
├── claude-code.json    # Claude Code 格式
├── codex.toml          # Codex CLI 格式
├── trae.json           # Trae 格式
└── hermes.yaml         # Hermes 格式
```

**或者**用通用 JSON,转换脚本处理。

### 3.5 对接方案速查

| Runtime | 推荐方式 | 说明 |
|---------|----------|------|
| Claude Code | 软链 skills/agents 到 .claude/ | Claude Code 强制目录 .claude/ |
| Codex CLI | 配置 external_dirs | 无需复制,直接读 `.agent/skills/` |
| Hermes Agent | 配置 external_dirs | 同 Codex |
| Trae | 软链 skills 和 AGENTS.md | Trae 强制目录 .trae/ |

## 4. 与资产层的关系

### 4.1 三层架构

```
┌──────────────────────────────────────────┐
│  知识库资产层(01-Skills/, 04-MCP/, ...)    │  唯一规范源,跨项目共享
└──────────────────┬───────────────────────┘
                   │ 复制/软链(由部署脚本完成)
                   ↓
┌──────────────────────────────────────────┐
│  项目级目录(.agent/)                       │  本项目需要的资产
└──────────────────┬───────────────────────┘
                   │ 复制/软链/外部引用(由 Runtime 对接完成)
                   ↓
┌──────────────────────────────────────────┐
│  Runtime 原生目录(.claude/ .codex/ ...)    │  Runtime 实际读取的位置
└──────────────────────────────────────────┘
```

> **可选层级**:简单项目可以跳过中间层,直接把资产层软链到 Runtime 原生目录。

### 4.2 何时在哪层编辑

| 动作 | 编辑层 |
|------|--------|
| 修改技能内容、新增技能 | 资产层 `01-Skills/` |
| 跨项目共享的 MCP 配置 | 资产层 `04-MCP/` |
| 本项目特定技能 | 项目级 `.agent/skills/` |
| 本项目 MCP 配置 | 项目级 `.agent/mcp/` |
| 项目说明、约定 | 项目根 `AGENTS.md` |
| 临时试验、不入库 | Runtime 原生目录(开发期) |

## 5. 部署脚本(自动对接 Runtime)

把"从资产层 → `.agent/` → Runtime 原生位置"的过程写成脚本:

### 5.1 PowerShell 版本

```powershell
# deploy-assets.ps1 — 项目级资产部署脚本
# 用途:把全局资产层复制到 .agent/,然后根据 Runtime 软链到原生目录
# 用法:.\scripts\deploy-assets.ps1 -Runtime claude-code

param(
    [Parameter(Mandatory)]
    [ValidateSet("claude-code", "codex", "hermes", "trae", "all")]
    [string]$Runtime,

    [string]$KbRoot = "$PSScriptRoot\..\..\00-AgentBase"
)

# 1. 确保 .agent/ 目录存在
$agentDirs = @("skills", "agents", "plugins", "prompts", "mcp", "hooks", "memory")
foreach ($dir in $agentDirs) {
    New-Item -ItemType Directory -Path ".\.agent\$dir" -Force | Out-Null
}

# 2. 从资产层复制全局技能到 .agent/skills/
Write-Host "`n[1/3] 同步资产层 → .agent/..."
$kbSkills = Join-Path $KbRoot "..\01-Skills"
Get-ChildItem -Path $kbSkills -Directory | ForEach-Object {
    $name = $_.Name
    $src = $_.FullName
    $dst = ".\.agent\skills\$name"
    if (-not (Test-Path $dst)) {
        Copy-Item -Recurse -Path $src -Destination $dst
        Write-Host "  ✅ 已部署: $name"
    } else {
        Write-Host "  — 已存在: $name"
    }
}

# 3. 根据 Runtime 软链 .agent/ → 原生目录
Write-Host "`n[2/3] 链接到 Runtime 原生目录..."

function Link-To-Runtime {
    param([string]$Source, [string]$Target)
    if (Test-Path $Target) {
        Remove-Item $Target -Recurse -Force
    }
    New-Item -ItemType Junction -Path $Target -Target $Source | Out-Null
    Write-Host "  🔗 $Target → $Source"
}

switch ($Runtime) {
    "claude-code" {
        New-Item -ItemType Directory -Path ".\.claude" -Force | Out-Null
        Link-To-Runtime "..\.agent\skills" ".\.claude\skills"
        Link-To-Runtime "..\.agent\agents" ".\.claude\agents"
        Link-To-Runtime "..\.agent\hooks"  ".\.claude\hooks"
        # .agent/mcp/claude-code.json → .mcp.json(项目根)
        if (Test-Path ".\.agent\mcp\claude-code.json") {
            Copy-Item -Force ".\.agent\mcp\claude-code.json" ".\.mcp.json"
            Write-Host "  🔗 .mcp.json ← .agent/mcp/claude-code.json"
        }
    }
    "codex" {
        New-Item -ItemType Directory -Path ".\.codex" -Force | Out-Null
        Link-To-Runtime "..\.agent\skills" ".\.codex\skills"
    }
    "hermes" {
        # Hermes 用配置引用,不需要软链
        Write-Host "  ℹ️  Hermes Agent:在配置中指定 external_dirs = .agent/skills"
    }
    "trae" {
        New-Item -ItemType Directory -Path ".\.trae" -Force | Out-Null
        Link-To-Runtime "..\.agent\skills" ".\.trae\skills"
        Link-To-Runtime "..\AGENTS.md"     ".\.trae\AGENTS.md"
    }
    "all" {
        & $PSCommandPath -Runtime claude-code
        & $PSCommandPath -Runtime codex
        & $PSCommandPath -Runtime hermes
        & $PSCommandPath -Runtime trae
    }
}

Write-Host "`n[3/3] 完成。AGENTS.md 在项目根,资产在 .agent/"
```

### 5.2 使用方式

```powershell
# 部署到 Claude Code
.\scripts\deploy-assets.ps1 -Runtime claude-code

# 同时部署到所有 Runtime
.\scripts\deploy-assets.ps1 -Runtime all
```

## 7. 从旧规范迁移

如果你已有项目用 `.trae/`、`.claude/` 等旧目录:

### 7.1 迁移步骤

```
1. 在项目根创建 .agent/ 目录结构
2. 把 .trae/skills/ 复制到 .agent/skills/
3. 把 .claude/skills/ 复制到 .agent/skills/(合并)
4. 把 .trae/AGENTS.md 复制到 AGENTS.md(项目根)
5. 在 AGENTS.md 第 6 节加上"资产在 .agent/"指向
6. 删掉 .trae/、.claude/ 等旧目录(或保留为软链到 .agent/)
7. .gitignore 改为忽略 .agent/(或按新规则)
8. 运行 deploy-assets.ps1 重新建立软链
9. 启动 Runtime 验证
```

### 7.2 不强制迁移

旧项目继续用 `.trae/`、`.claude/` 等**完全没问题**——本规范 v2.0 是新项目的默认,**不强制旧项目迁移**。
