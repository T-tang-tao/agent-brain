## 配置文件位置

| 配置层级 | 路径 | 作用域 |
|----------|------|--------|
| 用户级 | `~/.codex/config.toml` | 所有项目 |
| 项目级 | `.codex/config.toml` | 当前项目 |
| Profile | `~/.codex/<name>.config.toml` | 指定 Profile |
| 系统级 | `/etc/codex/config.toml` | 所有用户 |

所有路径受 `CODEX_HOME` 环境变量影响（默认 `~/.codex/`）。

## 配置优先级

6 级优先级，从高到低：

```
1. CLI 标志（--model, --sandbox 等）        ← 最高优先级
2. 项目配置（.codex/config.toml）
3. Profile 配置（~/.codex/<name>.config.toml）
4. 用户配置（~/.codex/config.toml）
5. 系统配置（/etc/codex/config.toml）
6. 内置默认值                               ← 最低优先级
```

> **类比**：像 CSS 优先级——内联样式（CLI 标志）覆盖页面样式（项目配置），页面样式覆盖全局样式（用户配置）。

## 核心配置项

```toml
# ~/.codex/config.toml 示例

# 模型设置
model = "gpt-5-codex"
model_reasoning_effort = "medium"    # minimal | low | medium | high
model_provider = "openai"            # 引用 [model_providers.openai]

# 审批与沙箱
approval_policy = "on-request"       # untrusted | on-request | never | granular
sandbox_mode = "workspace-write"     # read-only | workspace-write | danger-full-access
approvals_reviewer = "auto_review"   # 可选：自动审查

# 功能开关
web_search = true                    # 允许网络搜索

# 个性设置
personality = "concise"              # concise | verbose

# 禁用的工具
disable_response_storage = false     # 是否禁止存储响应
```

| 配置项 | 可选值 | 说明 |
|--------|--------|------|
| `model` | 模型名称 | 默认 `gpt-5-codex` |
| `model_reasoning_effort` | minimal/low/medium/high | 推理深度，越高越慢但更精确 |
| `model_provider` | provider 名称 | 引用 `[model_providers.<name>]` |
| `approval_policy` | untrusted/on-request/never/granular | 审批策略 |
| `sandbox_mode` | read-only/workspace-write/danger-full-access | 沙箱模式 |
| `approvals_reviewer` | auto_review | 自动审查模式（可选） |
| `web_search` | true/false | 是否启用网络搜索 |
| `personality` | concise/verbose | 回复风格 |

## Features 功能表

14 项功能开关，通过 `[features]` 表配置。每项功能有成熟度标签：

| 功能 | 默认 | 成熟度 | 说明 |
|------|------|--------|------|
| `apps` | false | experimental | 应用集成 |
| `codex_git_commit` | false | experimental | Git 提交自动化 |
| `hooks` | false | experimental | 生命周期钩子 |
| `fast_mode` | false | experimental | 快速模式 |
| `memories` | false | experimental | 记忆系统 |
| `multi_agent` | false | experimental | 多 Agent 协作 |
| `personality` | false | experimental | 个性化设置 |
| `shell_snapshot` | true | stable | Shell 环境快照 |
| `shell_tool` | true | stable | Shell 工具 |
| `unified_exec` | true | stable | 统一执行 |
| `undo` | true | stable | 撤销操作 |
| `web_search` | false | stable | 网络搜索 |

```toml
# 启用实验性功能
[features]
hooks = true
memories = true
multi_agent = true
```

> **注意**：标记为 `experimental` 的功能可能不稳定，生产环境慎用。

## Profile 配置

Profile 允许为不同场景预定义配置组：

```toml
# ~/.codex/review.config.toml — 代码审查 Profile
model = "gpt-5-codex"
model_reasoning_effort = "high"
approval_policy = "untrusted"
sandbox_mode = "read-only"
personality = "verbose"
```

使用 Profile：

```bash
codex --profile review
```

> **类比**：像 Git 的分支——不同 Profile 是不同配置分支，`--profile` 切换分支。

## Model Providers

Codex 支持多种模型 Provider，在 `[model_providers.<name>]` 中配置：

### OpenAI（默认）

```toml
[model_providers.openai]
name = "OpenAI"
base_url = "https://api.openai.com/v1"
env_key = "OPENAI_API_KEY"
wire_api = "responses"    # responses | chat
```

### Ollama（本地模型）

```toml
[model_providers.ollama]
name = "Ollama"
base_url = "http://localhost:11434/v1"
env_key = "OLLAMA_API_KEY"
wire_api = "chat"
```

### Mistral

```toml
[model_providers.mistral]
name = "Mistral"
base_url = "https://api.mistral.ai/v1"
env_key = "MISTRAL_API_KEY"
wire_api = "chat"
```

### Azure OpenAI

```toml
[model_providers.azure]
name = "Azure OpenAI"
base_url = "https://<your-resource>.openai.azure.com"
env_key = "AZURE_OPENAI_API_KEY"
wire_api = "chat"
```

### AWS Bedrock

```toml
[model_providers.bedrock]
name = "AWS Bedrock"
env_key = "AWS_BEDROCK_API_KEY"
wire_api = "bedrock"
```

| Provider | wire_api | 说明 |
|----------|----------|------|
| OpenAI | `responses` | 使用 Responses API（默认） |
| Ollama | `chat` | 本地模型，无需 API Key |
| Mistral | `chat` | Mistral AI |
| Azure | `chat` | Azure OpenAI Service |
| Bedrock | `bedrock` | AWS Bedrock |

## Shell 环境策略

`shell_environment_policy` 控制 Agent 执行命令时继承的环境变量：

```toml
[shell_environment_policy]
inherit = "core"              # none | core | all
exclude = ["SECRET_*"]        # 排除的环境变量模式
set = { CUSTOM_VAR = "value" } # 额外设置的环境变量
```

| inherit 值 | 说明 |
|------------|------|
| `none` | 不继承任何环境变量 |
| `core` | 仅继承核心变量（PATH, HOME 等） |
| `all` | 继承所有环境变量 |

> **安全建议**：生产环境用 `core`，避免泄露敏感环境变量。

## History 持久化

```toml
[history]
persistence = "save-all"      # save-all | none
```

| 值 | 说明 |
|----|------|
| `save-all` | 保存所有会话历史 |
| `none` | 不保存历史 |

## 完整配置示例

```toml
# ~/.codex/config.toml

model = "gpt-5-codex"
model_reasoning_effort = "medium"
model_provider = "openai"
approval_policy = "on-request"
sandbox_mode = "workspace-write"
web_search = true
personality = "concise"

[features]
hooks = true
memories = true

[model_providers.openai]
name = "OpenAI"
base_url = "https://api.openai.com/v1"
env_key = "OPENAI_API_KEY"
wire_api = "responses"

[shell_environment_policy]
inherit = "core"
exclude = ["SECRET_*", "TOKEN_*"]

[history]
persistence = "save-all"
```
