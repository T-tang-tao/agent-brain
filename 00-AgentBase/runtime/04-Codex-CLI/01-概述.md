## 这是什么

Codex CLI 是 OpenAI 开发的开源本地编码 Agent。它用 Rust 编写，在终端中以 TUI（终端用户界面）运行，能够理解代码库、读写文件、执行命令、运行测试，并通过沙箱隔离确保操作安全。

核心特征：

- **本地终端运行**：Rust 原生二进制，启动快，TUI 交互流畅
- **沙箱隔离**：所有操作在沙箱中执行，读写受控，网络可隔离
- **审批策略**：四种审批模式，从全自动到手动确认，灵活控制
- **多模型支持**：默认 OpenAI 模型，可配置 Ollama、Mistral、Azure、Bedrock 等
- **MCP 扩展**：通过 MCP 协议连接外部工具服务器
- **AGENTS.md 上下文**：通过 AGENTS.md 文件注入项目约定和工作流
- **Skills 技能系统**：SKILL.md 格式的渐进式披露技能，按需加载

## 类比

大多数编码助手像"打字员"——你问什么它答什么。Codex CLI 像"安全工程师"——它会在沙箱里试验，提交前等你审批，出错能撤销，不碰不该碰的文件。

## 核心特性

| 特性 | 说明 |
|------|------|
| **沙箱执行** | 三种沙箱模式：只读、工作区写入、完全访问。macOS 用 Seatbelt，Linux 用 bwrap+seccomp，Windows 原生隔离 |
| **审批策略** | untrusted（全确认）、on-request（按需确认）、never（全自动）、granular（细粒度）。支持 auto-review 自动审查 |
| **网络隔离** | network_proxy 功能，域名白名单，DNS 重绑定保护。默认阻断网络访问，按需放行 |
| **配置分层** | 6 级配置优先级：CLI 标志 > 项目配置 > Profile > 用户配置 > 系统配置 > 默认值 |
| **多模型 Provider** | OpenAI（默认）、Ollama（本地）、Mistral、Azure OpenAI、AWS Bedrock |
| **MCP 集成** | STDIO + HTTP 两种传输，OAuth 认证，工具过滤，插件提供 MCP |
| **AGENTS.md** | 全局 `~/.codex/AGENTS.md` + 项目 `AGENTS.md`，注入项目约定和工作流 |
| **Skills 系统** | SKILL.md 格式，YAML frontmatter + Markdown 正文，渐进式披露，全局/项目两种作用域 |
| **14 项 Features** | apps、codex_git_commit、hooks、fast_mode、memories、multi_agent、personality 等，带成熟度标签 |
| **Dev Containers** | 支持在 Dev Containers 中运行，提供更强的隔离环境 |

## 设计哲学

Codex CLI 的设计围绕一个核心问题：**如何让 Agent 安全地操作代码？**

答案是两层防御：

```
┌─────────────────────────────────────────────┐
│  第一层：沙箱（Sandbox）                      │
│  控制操作能力——能读写什么、能否联网            │
│  ┌─────────────────────────────────────┐    │
│  │  第二层：审批（Approval）             │    │
│  │  控制操作确认——哪些操作需要人批准      │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

- **沙箱**决定 Agent **能做什么**（能力边界）
- **审批**决定 Agent **做了之后需不需要你同意**（确认机制）

两者正交组合，覆盖从"全自动但隔离"到"全权限但手动确认"的所有场景。

## 核心命令

| 命令 | 作用 |
|------|------|
| `codex` | 启动交互式 TUI 对话 |
| `codex "任务描述"` | 直接执行单次任务 |
| `codex --profile <name>` | 使用指定 Profile 配置 |
| `codex --model <model>` | 指定模型 |
| `codex --sandbox <mode>` | 指定沙箱模式 |
| `codex --approval <policy>` | 指定审批策略 |
| `codex mcp add <name>` | 添加 MCP 服务器 |
| `codex mcp list` | 列出已配置的 MCP 服务器 |
| `codex --full-auto` | 全自动模式（sandbox=workspace-write, approval=never） |
