## 自定义构建顺序

Codex 推荐按以下顺序构建自定义能力，从轻量到重量：

```
1. AGENTS.md        ← 零成本：写个项目说明文件
      ↓
2. Skills/Plugins   ← 低成本：写个 SKILL.md 操作手册
      ↓
3. MCP Servers      ← 中成本：部署外部工具服务器
      ↓
4. Subagents        ← 高成本：定义隔离子 Agent
```

> **原则**：先用轻量手段解决问题，不够再升级。不要一上来就写 MCP 服务器。

## AGENTS.md

AGENTS.md 是 Codex 的项目上下文文件，告诉 Agent "这个项目的约定是什么"。

### 两层 AGENTS.md

| 层级 | 路径 | 作用域 |
|------|------|--------|
| 全局 | `~/.codex/AGENTS.md` | 所有项目 |
| 项目 | `<项目根>/AGENTS.md` | 当前项目 |

两者合并使用：全局定义通用偏好，项目定义项目特定约定。

### AGENTS.md 写什么

```markdown
# 项目: My App

## 技术栈
- 前端: React 18 + TypeScript
- 后端: Node.js + Express
- 数据库: PostgreSQL

## 代码约定
- 使用 ESLint + Prettier
- 提交前必须通过 lint
- 测试用 Vitest

## 常用命令
- `npm run dev` — 启动开发服务器
- `npm test` — 运行测试
- `npm run lint` — 代码检查

## 注意事项
- 不要修改 migrations/ 下的已有文件
- API 响应格式统一为 { data, error, meta }
```

> **类比**：像给新同事写的项目入门文档——告诉他项目用什么、怎么跑、什么不能碰。

### AGENTS.md 与本知识库的关系

本知识库的 `00-AgentBase/AGENTS.md` 就是 Codex 全局 AGENTS.md 的知识库版本——定义 Schema 和管理规范，而根目录 `AGENTS.md` 是部署、编排与管理指南。两者概念一致，只是应用场景不同。

## Skills 技能系统

Skill 是 Codex 的按需加载操作手册，通过 SKILL.md 文件定义。

### SKILL.md 格式

```markdown
---
name: my-skill
description: Parse, validate, and transform CSV files. Use when importing data from CSV, exporting reports, or fixing encoding issues.
version: 1.0.0
---

# CSV 处理技能

## When to Use
- 导入 CSV 数据
- 导出 CSV 报告
- 修复 CSV 编码问题

## Procedure
1. 检测文件编码
2. 解析 CSV 头部
3. 验证数据类型
4. 执行转换
5. 输出结果

## Pitfalls
- BOM 头导致第一个列名异常
- 引号内的换行符需要特殊处理
```

### YAML Frontmatter 字段

| 字段 | 必需 | 说明 |
|------|------|------|
| `name` | 是 | 技能名称，小写+连字符 |
| `description` | 是 | 功能描述，**给模型读的触发器** |
| `version` | 否 | 版本号 |

> **关键**：`description` 不是给人读的说明，是给模型读的触发器。要写具体场景，不要写泛泛功能。

### 渐进式披露

Skill 采用三级渐进式披露，按需加载：

```
┌─────────────────────────────────────────────────────┐
│  Level 0: 元数据                                     │
│  始终加载 · ~100 token · name + description          │
│  Agent 从目录中判断哪些技能可能相关                     │
├─────────────────────────────────────────────────────┤
│  Level 1: 完整指令                                   │
│  按需加载 · < 5,000 token · SKILL.md 的 Markdown 正文 │
│  Agent 判断某个技能相关时，才读取完整指令                │
├─────────────────────────────────────────────────────┤
│  Level 2: 支持文件                                   │
│  按需加载 · 大小不限 · references/templates/scripts   │
│  只有 SKILL.md 中明确引用时，Agent 才读取这些文件       │
└─────────────────────────────────────────────────────┘
```

> **含义**：Skill 能否被使用，完全取决于 Level 0——那约 100 个 token 的 name 和 description。description 模糊，Skill 永远不会被调用。

### 两种作用域

| 作用域 | 路径 | 适用 |
|--------|------|------|
| 全局技能 | `~/.agents/skills/` | 所有项目可用 |
| 项目技能 | `<项目>/.agents/skills/` | 当前项目专用 |

```
# 全局技能
~/.agents/skills/
├── csv-handler/
│   └── SKILL.md
└── code-reviewer/
    └── SKILL.md

# 项目技能
<project>/.agents/skills/
├── deploy/
│   ├── SKILL.md
│   └── references/
│       └── deploy-steps.md
```

### Skill 目录结构

```
my-skill/
├── SKILL.md           # 主指令文件（必需）
├── templates/         # 模板文件（可选）
│   └── report.md
├── references/        # 参考资料（可选）
│   └── style-guide.md
├── examples/          # 示例输出（可选）
│   └── sample.md
└── scripts/           # 可执行脚本（可选）
    └── validate.sh
```

### 过程与知识分离

| 类型 | 属于 | 放哪里 | 类比 |
|------|------|--------|------|
| **过程**（Process） | Agent 应遵循的有序步骤 | SKILL.md 正文 | 食谱："第一步打鸡蛋" |
| **知识**（Context） | 背景信息、规则、领域知识 | references/ 目录 | 百科："面粉的品种" |

> **原则**：SKILL.md 要写得像食谱，不像百科全书。

## Subagents

Subagents 是 Codex 的多 Agent 协作机制（需启用 `multi_agent` feature）。

```
┌─────────────┐
│  主 Agent   │ ────委托────→ ┌──────────────┐
│  (Codex)    │               │  子 Agent    │
│             │ ←──结果────── │  (隔离上下文) │
└─────────────┘               └──────────────┘
```

### 使用场景

| 场景 | 说明 |
|------|------|
| 并行任务 | 多个子 Agent 同时处理独立任务 |
| 上下文隔离 | 子任务不污染主对话上下文 |
| 专业化 | 不同子 Agent 配置不同工具集和指令 |

### 配置

```toml
[features]
multi_agent = true    # 启用多 Agent 功能（实验性）
```

> **注意**：`multi_agent` 为实验性功能，API 可能变化。

## 自定义层级总结

| 层级 | 成本 | 灵活度 | 典型场景 |
|------|------|--------|----------|
| AGENTS.md | 零 | 低 | 项目约定、常用命令、注意事项 |
| Skills | 低 | 中 | 可复用的任务流程（CSV 处理、代码审查） |
| Plugins | 低 | 中 | 打包分发 Skills + MCP 配置 |
| MCP | 中 | 高 | 外部工具集成（数据库、API） |
| Subagents | 高 | 最高 | 并行任务、上下文隔离 |
