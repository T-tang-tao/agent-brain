## 什么是子代理

子代理(Subagent)是在**独立上下文窗口**中运行的专门化 AI 助手:

- 拥有自定义 system prompt(独立人格)
- 拥有独立的工具访问权限(可白名单/黑名单)
- 拥有独立的权限模式(可与父级不同)
- 父代理委派任务后,子代理在隔离环境中完成

> 类比:父代理像"项目经理"——分解任务、分配给专家;子代理像"各专业工程师"——在独立工位完成本职工作,只汇报结果,不污染项目经理的上下文。

## 内置子代理

Claude Code 预装了多个专用子代理:

| 子代理 | 模型 | 工具权限 | 用途 |
|--------|------|----------|------|
| **Explore** | 继承父级(上限 Opus) | 只读 | 快速代码搜索与探索 |
| **Plan** | 继承父级 | 只读 | plan 模式研究分析 |
| **general-purpose** | 继承父级 | 全部工具 | 复杂多步任务 |
| **statusline-setup** | Sonnet | 配置类 | 配置状态栏 |
| **claude-code-guide** | Haiku | 只读 | 回答 Claude Code 使用问题 |

### Explore 与 Plan 的加速

为提升速度,Explore 和 Plan 两个子代理**跳过 CLAUDE.md 加载和 git status 检查**——它们只做搜索和分析,不需要项目上下文,从而更快返回结果。

> 类比:像侦察兵——轻装上阵不带辎重(项目上下文),快速侦察后回报,主力部队(主代理)再带完整装备行动。

## 子代理作用域

子代理定义文件按优先级从高到低加载:

| 优先级 | 位置 | 说明 |
|--------|------|------|
| 1(最高) | Managed policy | 企业级强制子代理 |
| 2 | `--agents` CLI 标志 | 命令行临时指定 |
| 3 | `.claude/agents/` | 项目级(提交到 git,团队共享) |
| 4 | `~/.claude/agents/` | 用户级(跨所有项目) |
| 5(最低) | Plugin agents | 插件提供的子代理 |

> 类比:像组织架构——总部策略 > 临时委派 > 部门规章 > 个人习惯 > 外包供应商,优先级从高到低。

## 子代理文件格式

每个子代理是一个 Markdown 文件,带 YAML frontmatter:

```markdown
---
name: code-reviewer
description: 审查代码变更,关注安全和性能
tools: [Read, Grep, Glob]          # 工具白名单
disallowedTools: [Bash]            # 工具黑名单
model: sonnet                      # 指定模型
permissionMode: plan               # 权限模式
mcpServers: [github]               # 允许的 MCP 服务器
hooks: [pre-commit-lint]           # 关联 Hooks
maxTurns: 20                       # 最大轮次
skills: [review-checklist]         # 预加载技能
initialPrompt: "开始审查..."        # 初始提示
memory: true                       # 持久记忆
effort: high                       # 推理强度
background: true                   # 后台运行
isolation: strict                  # 隔离级别
color: blue                        # 终端显示颜色
---

你是资深代码审查专家。重点关注:
1. 安全漏洞
2. 性能瓶颈
3. 代码可维护性
```

frontmatter 字段全部可选,未指定则继承父级设置。

## 模型选择

| 方式 | 说明 |
|------|------|
| 继承父级 | 默认行为,子代理用与父代理相同的模型 |
| 显式指定 | 在 frontmatter 设 `model: haiku/sonnet/opus` |

> 类比:像给子代理分配"职级"——简单搜索用实习生(Haiku),复杂审查用专家(Opus),按任务难度匹配,控制成本。

## 工具控制

两种机制控制子代理可用工具:

| 机制 | 字段 | 语义 | 效果 |
|------|------|------|------|
| 白名单 | `tools` | 只允许这些工具 | 缩小范围,提升安全 |
| 黑名单 | `disallowedTools` | 禁止这些工具 | 排除危险操作 |

**当两者同时设置时,黑名单(disallowedTools)优先级更高**——即使工具在白名单中,只要出现在黑名单就会被禁止。

> 类比:白名单像"通行证"(有证才能进),黑名单像"黑名单"(在册就禁入)。两者冲突时,安全优先(黑名单赢)。

## 权限模式

子代理可以继承父级权限模式,也可独立设置:

```yaml
permissionMode: inherit    # 继承父级
permissionMode: plan       # 只读分析模式
permissionMode: default    # 手动确认模式
permissionMode: acceptEdits  # 自动编辑模式
```

这让"只读探索子代理"和"可写执行子代理"可以共存于同一会话。

## 预加载技能

子代理可在 frontmatter 中预加载技能:

```yaml
skills:
  - review-checklist
  - security-audit
```

技能在子代理启动时自动加载,无需运行时再调用,确保子代理一开始就"装备齐全"。

## 持久记忆

启用 `memory: true` 后,子代理拥有自己的持久记忆:

- 跨会话保存学到的经验
- 与主代理记忆隔离
- 适合长期运行的专业子代理积累领域知识

## 前台 vs 后台

| 模式 | 触发 | 行为 |
|------|------|------|
| **后台**(默认) | v2.1.198+ 默认 | 子代理独立运行,父代理可继续工作 |
| **前台** | 需要结果才能继续时 | 父代理等待子代理完成后才继续 |

> 类比:后台像"异步任务"——派出去就干别的;前台像"同步调用"——必须等结果才能下一步。

## Fork 当前对话

Fork(分叉)创建一个继承父级**完整对话历史**的子代理:

- 继承父代理全部上下文(不是空白启动)
- 始终以后台方式运行
- 适合"在当前对话基础上探索分支方案"

> 类比:像 Git 的 branch——从当前 commit 分叉出去,在新分支上实验,不影响主线。

## 常见模式

| 模式 | 说明 | 示例 |
|------|------|------|
| **隔离高量操作** | 把产生大量输出的操作隔离到子代理 | 日志分析、测试运行 |
| **并行研究** | 同时派多个子代理调研不同方向 | 对比 3 个库的优劣 |
| **链式子代理** | 一个子代理的输出喂给下一个 | 调研→设计→实现 |
| **嵌套子代理** | 子代理再生子代理 | 复杂任务层层分解 |

## 长时运行与禁用

- **自动压缩**:长时间运行的子代理在上下文接近上限时自动总结早期对话,保留关键信息继续运行,无需人工干预
- **禁用特定子代理**:在 `permissions.deny` 中按名添加,如 `Agent(code-reviewer)`
- **禁用全部子代理**:直接禁用 Agent 工具,阻止所有子代理生成
