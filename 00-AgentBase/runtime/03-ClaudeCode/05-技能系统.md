## 这是什么

技能(Skill)是扩展 Claude Code 能力的 `SKILL.md` 文件。它不是代码,而是指导 Claude 完成特定任务的"操作手册"。Claude 在相关时自动使用,或通过 `/skill-name` 直接调用。

**自定义命令已合并进技能**:`.claude/commands/deploy.md` 和 `.claude/skills/deploy/SKILL.md` 都创建 `/deploy`,工作方式相同。技能增加了支持文件、frontmatter 控制和自动加载等特性。

关键特征:

- 兼容 [agentskills.io](https://agentskills.io) 开放标准(跨多种 AI 工具通用)
- Claude Code 扩展了标准:调用控制、子代理执行、动态上下文注入
- 技能内容**按需加载**,不用时不占 token

> **类比**:技能像"按需查阅的 SOP 手册"——书架上摆着目录(只看描述),需要时才抽出完整内容,用完放回。与 CLAUDE.md 不同,技能不用时几乎零成本。

### 何时创建技能

| 信号 | 说明 |
|------|------|
| 反复粘贴相同指令 | 同样的步骤重复输入,应固化成技能 |
| CLAUDE.md 段落变成流程 | 当事实描述演变成操作步骤时,迁移到技能 |

## 技能位置

存储位置决定谁能使用,优先级从高到低:

| 位置 | 路径 | 适用于 |
|------|------|--------|
| **Enterprise** | 托管设置(managed settings) | 组织所有用户 |
| **Personal** | `~/.claude/skills/<skill-name>/SKILL.md` | 你的所有项目 |
| **Project** | `.claude/skills/<skill-name>/SKILL.md` | 仅本项目 |
| **Plugin** | `<plugin>/skills/<skill-name>/SKILL.md` | 插件启用处 |

**优先级**:Enterprise > Personal > Project。任一层级的同名技能覆盖同名的内置技能。Plugin 技能使用 `plugin-name:skill-name` 命名空间,不与其他层级冲突。

> **类比**:像"公司政策 > 个人习惯 > 项目约定"——高优先级覆盖低优先级,但插件技能有自己的命名空间互不干扰。

## SKILL.md 格式

技能核心是 `SKILL.md` 文件,由 YAML frontmatter 和 Markdown 正文组成:

```markdown
---
name: my-skill
description: 简短描述技能做什么
when_to_use: 触发短语或示例请求
allowed-tools: Read Grep
disable-model-invocation: false
context: inline
---
# 技能标题

## 操作步骤
1. 第一步
2. 第二步
```

### Frontmatter 字段

| 字段 | 必需 | 说明 |
|------|------|------|
| `name` | 否 | 显示名称,默认取目录名 |
| `description` | 推荐 | 技能用途,Claude 据此决定何时使用(截断 1536 字符) |
| `when_to_use` | 否 | 附加触发上下文,追加到 description |
| `argument-hint` | 否 | 自动补全时显示的参数提示,如 `[issue-number]` |
| `arguments` | 否 | 命名位置参数,用于 `$name` 替换 |
| `disable-model-invocation` | 否 | 设为 `true` 时仅手动 `/skill-name` 调用 |
| `allowed-tools` | 否 | 预批准工具列表 |
| `context` | 否 | `inline`(默认,主对话内)或 `fork`(子代理中) |
| `model` | 否 | 指定运行模型 |

> 所有字段可选,仅 `description` 推荐填写,让 Claude 知道何时使用。

### 命令名来源

技能的斜杠命令名**默认取目录名**,而非 frontmatter 的 `name` 字段:

```
.claude/skills/summarize-changes/SKILL.md  →  /summarize-changes
```

`name` 字段仅用于技能列表中的显示名称。

## 动态上下文注入

使用 `` !`command` `` 语法在技能内容中注入命令输出——Claude Code 在 Claude 看到内容前先运行命令并替换该行:

```markdown
## 当前变更

!`git diff HEAD`

## 指令

总结上述变更,列出风险点。
```

> **类比**:像"信封里预填了实时数据"——Claude 打开技能时,git diff 的结果已经内联好了,无需再自己去查。

## 支持文件

技能目录可包含除 `SKILL.md` 外的辅助文件:

```
my-skill/
├── SKILL.md          # 主指令(必需)
├── template.md       # 模板供 Claude 填充
├── examples/
│   └── sample.md     # 示例输出
└── scripts/
    └── validate.sh   # 可执行脚本
```

在 `SKILL.md` 中引用这些文件,让 Claude 知道它们的存在和加载时机。

## 调用控制

| 设置 | 行为 |
|------|------|
| `disable-model-invocation: true` | 仅手动 `/skill-name` 调用,Claude 不会自动触发 |
| 默认(未设或 false) | Claude 自动判断 + 手动均可 |

## 预批准工具

`allowed-tools` 字段预批准技能使用的工具,无需逐次确认:

```yaml
allowed-tools: Read Grep Bash(npm run test *)
```

## 内置技能

Claude Code 自带一组内置技能,每个会话默认可用(可用 `disableBundledSkills` 设置禁用):

| 技能 | 用途 |
|------|------|
| `/code-review` | 代码审查 |
| `/batch` | 批量处理 |
| `/debug` | 调试辅助 |
| `/loop` | 循环执行 |
| `/claude-api` | Claude API 相关操作 |
| `/run` | 启动并驱动应用查看变更效果 |
| `/verify` | 构建运行应用确认变更正确 |
| `/run-skill-generator` | 教 `/run` 和 `/verify` 如何启动项目 |

> 内置技能是基于 prompt 的——给 Claude 详细指令,让它用工具编排工作,而非固定逻辑。

## 实时变更检测

Claude Code 监视技能目录的文件变更:

| 行为 | 说明 |
|------|------|
| **热生效** | 添加、编辑、删除技能在当前会话内即时生效,无需重启 |
| **新建顶层目录** | 会话启动时不存在的 skills 目录需重启才能监视 |
| **范围** | 仅 `SKILL.md` 文本;插件文件夹的 hooks/MCP/agents 变更需 `/reload-plugins` |

## 嵌套目录发现

Monorepo 的各 package 可拥有自己的技能:

- 项目根和所有父级目录的 `.claude/skills/` 自动加载
- 编辑子目录文件时,该子目录的 `.claude/skills/` 按需发现
- 同名技能均保留,嵌套的以目录限定名展示(如 `apps/web:deploy`)

## 技能 vs CLAUDE.md

| 维度 | 技能(Skill) | CLAUDE.md |
|------|--------------|-----------|
| **加载时机** | 按需调用时加载 | 每次会话启动加载 |
| **token 成本** | 不用时近零 | 每次会话持续消耗 |
| **触发方式** | 斜杠命令 + Claude 自动 | 始终在上下文中 |
| **适用内容** | 操作流程、多步过程 | 事实、约定、架构决策 |
| **内容生命周期** | 加载后跨轮次留在上下文 | 始终在上下文 |

> **核心区别**:CLAUDE.md 是"每次都带的备忘录",技能是"需要时才翻的手册"。当 CLAUDE.md 段落从事实变成流程时,就该迁移到技能。

### 技能内容生命周期

技能一旦加载,其内容**跨轮次留在上下文中**——每一行都是持续的 token 成本。因此技能正文应简洁:说做什么,而非叙述怎么做或为什么。
