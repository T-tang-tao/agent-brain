## 权限模式

Claude Code 提供六种权限模式,控制 Claude 编辑文件或执行命令前是否需要确认:

| 模式 | 无需确认的操作 | 适用场景 |
|------|---------------|----------|
| **default**(手动) | 仅读取 | 入门使用、敏感操作 |
| **acceptEdits** | 读取 + 文件编辑 + 常见文件系统命令(mkdir/touch/mv/cp 等) | 迭代修改代码时 |
| **plan** | 仅读取(分析后提出方案,不修改) | 探索代码库后再决定 |
| **auto** | 全部操作,但有后台安全分类器审核 | 长任务、减少确认疲劳 |
| **dontAsk** | 仅预批准的工具 | 锁定的 CI 和脚本 |
| **bypassPermissions** | 全部操作(无限制) | 仅限隔离容器和 VM |

> **类比**:权限模式像"自动驾驶级别"——从 L0(全手动)到 L5(全自动),按信任程度选择。plan 模式像"先看路况再出发",auto 模式像"有副驾监督的自动驾驶"。

## 模式切换

| 方式 | 操作 |
|------|------|
| 会话中 | `Shift+Tab` 循环:default → acceptEdits → plan(其他模式按需出现) |
| 启动时 | `claude --permission-mode plan` |
| 持久默认 | settings.json 中 `"permissions": {"defaultMode": "acceptEdits"}` |
| 单次提示 | 输入 `/plan` 前缀 |

`auto` 和 `bypassPermissions` 需要额外条件才出现在循环中。`dontAsk` 不出现在循环中,只能通过 `--permission-mode dontAsk` 设置。

## acceptEdits 模式

自动批准文件编辑和常见文件系统命令:

- 文件编辑(Edit/Write)
- Bash 命令:`mkdir`、`touch`、`rm`、`rmdir`、`mv`、`cp`、`sed`
- PowerShell 命令:`Set-Content`、`Add-Content`、`Clear-Content`、`Remove-Item`
- 安全前缀:`LANG=C`、`NO_COLOR=1`、`timeout`、`nice`、`nohup`

仅限工作目录和 `additionalDirectories` 内的路径。受保护路径和目录外操作仍需确认。

## plan 模式

Claude 研究代码库并提出方案,不做任何修改:

- 读取文件、运行探索性命令
- 写出计划方案
- 呈现方案后选择:批准并切 auto / 批准并 acceptEdits / 批准并手动审查 / 继续规划 / 用 Ultraplan 精炼
- `Ctrl+G` 可在文本编辑器中直接编辑方案

## auto 模式

独立分类器模型审核每个操作,阻止超出请求范围、针对未知基础设施、或由恶意内容驱动的操作。

**要求**:所有付费计划;Team/Enterprise 需管理员启用;需支持的模型(Opus 4.6+/Sonnet 4.6+ on API;Sonnet 5/Opus 4.7/4.8 on Bedrock/Vertex/Foundry)。

**默认阻止**:

| 类别 | 示例 |
|------|------|
| 下载执行代码 | `curl \| bash` |
| 发送敏感数据到外部 | 向未知端点上传文件 |
| 生产部署和迁移 | 直接部署到生产环境 |
| 云存储批量删除 | 删除 S3 桶内容 |
| 授予权限 | 修改 IAM 或 repo 权限 |
| 修改共享基础设施 | 改 DNS、TLS 证书 |
| 不可逆文件销毁 | 删除会话前已存在的文件 |
| Git 危险操作 | force push、push 到 main、git reset --hard、git clean -fd |
| IaC 销毁 | terraform destroy、pulumi destroy、cdk destroy |

> auto 模式信任工作目录和会话开始时已配置的远程仓库。会话中新增的 remote 不被信任。

## 权限规则

在模式之上叠加 `allow` / `deny` / `ask` 规则,预批准或阻止特定工具:

```json
{
  "permissions": {
    "allow": ["Bash(npm run lint)", "Bash(npm run test *)", "Read(~/.zshrc)"],
    "deny": ["Bash(curl *)", "Read(./.env)", "Read(./.env.*)", "Read(./secrets/**)"]
  }
}
```

| 规则类型 | 作用 |
|----------|------|
| `allow` | 匹配的操作自动批准,无需确认 |
| `deny` | 匹配的操作永远阻止(所有模式,含 bypassPermissions) |
| `ask` | 匹配的操作强制确认(即使 auto 模式也弹窗) |

deny 规则在所有模式中生效(包括 `bypassPermissions`)。allow 规则在 `bypassPermissions` 模式无意义(已全部批准)。

## 权限规则语法

| 规则格式 | 适用工具 | 说明 |
|----------|----------|------|
| `Bash(npm run *)` | Bash, Monitor | 命令模式匹配 |
| `PowerShell(Get-ChildItem *)` | PowerShell | 命令模式匹配 |
| `Read(~/secrets/**)` | Read, Grep, Glob, LSP | 路径模式匹配 |
| `Edit(/src/**)` | Edit, Write, NotebookEdit | 路径模式匹配 |
| `Skill(deploy *)` | Skill | 技能名匹配 |
| `Agent(Explore)` | Agent | 子代理类型匹配 |
| `WebFetch(domain:example.com)` | WebFetch | 域名匹配 |
| `WebSearch` | WebSearch | 无参数,整体允许/拒绝 |

> `Edit(...)` 允许规则同时授予同路径的读权限,无需额外 `Read(...)` 规则。

## 受保护路径

无论什么模式(除 `bypassPermissions`),以下路径的写操作**永不自动批准**:

- `.git/` 目录(Git 仓库状态)
- Claude Code 自身配置目录(`.claude/`)
- 工作目录外的路径

> **类比**:受保护路径像"保险箱"——即使你授权了"自动整理房间",保险箱的钥匙也不会自动交出去。

## 工作目录

Claude Code 的工作目录概念:

| 概念 | 说明 |
|------|------|
| 启动目录 | `claude` 启动时的当前目录 |
| 额外目录 | `--add-dir`、`/add-dir` 或 `additionalDirectories` 设置添加的目录 |
| 文件访问范围 | 启动目录 + 额外目录 |

额外目录授予文件访问权限,但**不加载**该目录的 `.claude/` 配置(技能是例外)。

## 沙箱隔离

Windows 原生不支持沙箱。WSL 2 支持沙箱化命令执行:

| 选项 | 沙箱支持 | 适用 |
|------|----------|------|
| 原生 Windows | 不支持 | Windows 原生项目 |
| WSL 2 | 支持 | Linux 工具链或沙箱执行 |
| WSL 1 | 不支持 | WSL 2 不可用时 |

## 托管安全设置

企业管理员可强制:

| 设置 | 作用 |
|------|------|
| `allowManagedPermissionRulesOnly` | 阻止用户/项目定义权限规则 |
| `disableAutoMode` | 禁用 auto 模式 |
| `allowManagedHooksOnly` | 仅允许托管 Hooks |
| `allowManagedMcpServersOnly` | 仅允许白名单 MCP 服务器 |
| `requiredMinimumVersion` / `requiredMaximumVersion` | 版本范围限制 |

> 安全字段在 managed settings 中容错处理:无效值被剥离而非整体拒绝,确保单个错误不会使整个策略失效。
