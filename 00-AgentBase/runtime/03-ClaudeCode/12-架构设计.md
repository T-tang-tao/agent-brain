## 系统概览

```
入口点
  终端 CLI | VS Code 扩展 | JetBrains 插件 | 桌面应用 | Web | 移动端
                    ↓
  Claude Code 引擎
    ├── 系统提示组装(CLAUDE.md + Rules + Skills + Auto Memory)
    ├── Agentic 循环(规划 → 工具调用 → 验证 → 继续/停止)
    ├── 工具调度(Built-in + MCP + Skills)
    ├── 子代理委派(Explore / Plan / Custom)
    ├── 权限检查(模式 + 规则 + Hooks)
    └── 上下文管理(压缩 / 保留 / 缓存)
                    ↓
  响应输出 + 会话持久化
```

> 一个引擎服务所有入口点——多表面的差异在进入核心前被抹平。

## Agentic 循环

Claude Code 的核心是 Agentic 循环,不是简单的一问一答:

```
用户输入
  → 系统提示组装
  → 模型推理(决定下一步动作)
  → 权限检查(是否允许?)
  → 工具执行(Bash/Read/Edit/MCP...)
  → 结果反馈给模型
  → 继续推理 or 结束响应
  → 显示 + 持久化
```

| 阶段 | 说明 |
|------|------|
| **规划** | 模型分析任务,决定使用哪些工具、按什么顺序 |
| **执行** | 调用工具,执行操作(Bash 命令、文件编辑、搜索等) |
| **验证** | 检查工具输出,判断是否需要继续或已完成 |
| **迭代** | 未完成则继续循环,完成则结束响应 |

> **类比**:Agentic 循环像"有经验的工程师"——拿到需求后不会只做一步,而是规划→执行→检查→调整,直到完成。

## 系统提示组装

每次会话启动时,系统提示从多个来源组装:

```
系统提示 = 内置指令
         + CLAUDE.md(多级:Managed → User → Project → Local)
         + .claude/rules/*.md(路径特定规则按需加载)
         + Auto Memory(前 200 行 / 25KB)
         + 工具定义(Built-in + MCP)
         + 技能目录(描述摘要,不展开内容)
         + 上下文信息(工作目录、Git 状态、环境)
```

| 来源 | 加载时机 | 消耗 |
|------|----------|------|
| CLAUDE.md | 每次会话启动 | 全文进入上下文 |
| .claude/rules/ | 启动时(无条件)或文件匹配时(路径特定) | 仅匹配时消耗 |
| Auto Memory | 每次会话启动 | 前 200 行 / 25KB |
| 技能目录 | 每次会话启动 | 仅描述摘要(~1.5K 字符/技能) |
| 技能内容 | 调用时加载 | 加载后跨轮次驻留 |

## 上下文管理

Claude Code 管理长对话的上下文窗口:

| 机制 | 作用 |
|------|------|
| **上下文窗口可视化** | 展示 CLAUDE.md、对话、工具输出在上下文中的位置 |
| **自动压缩** | 上下文接近上限时自动压缩(PreCompact/PostCompact Hooks) |
| **手动压缩** | `/compact` 手动触发压缩 |
| **会话恢复** | `claude -c` / `claude -r` 恢复历史会话 |
| **子代理隔离** | 子代理在独立上下文窗口运行,不污染主对话 |

> **类比**:上下文窗口像"工作台"——CLAUDE.md 是钉在墙上的备忘录,对话是桌上的文件,工具输出是刚打印的资料。桌子空间有限,旧文件需要归档(压缩)才能放新的。

## 工具调度

```
模型请求工具调用
  → 权限检查(模式 + allow/deny 规则)
  → PreToolUse Hook(可拦截)
  → 工具执行
  → PostToolUse Hook(可后处理)
  → 结果返回模型
```

| 工具来源 | 发现方式 |
|----------|----------|
| Built-in | 内置,始终可用 |
| MCP | 连接时自动发现,Tool Search 延迟加载 |
| Skills | 目录扫描,`SKILL.md` 作为入口 |

## 子代理委派

当任务需要隔离上下文时,Claude 委派给子代理:

```
主对话
  ├── 检测到可委派任务
  ├── 选择子代理(按 description 匹配)
  ├── 生成子代理(独立上下文窗口)
  │   ├── 加载子代理系统提示
  │   ├── 应用工具限制(tools / disallowedTools)
  │   ├── 应用权限模式
  │   └── 独立执行任务循环
  └── 接收子代理返回的摘要结果
```

主对话只看到子代理的**最终摘要**,不看中间工具调用——节省上下文。

## 数据流

### CLI 会话

```
用户输入
  → UserPromptSubmit Hook(可修改/阻止)
  → 系统提示 + 上下文组装
  → 模型 API 调用
  → 工具调用循环(权限 → 执行 → Hook → 结果)
  → Stop Hook(可阻止结束/添加上下文)
  → 响应显示
  → 会话持久化
```

### 后台代理

```
claude --bg "task"
  → 创建后台会话(独立进程)
  → 通过 claude agents 监控
  → claude attach <id> 接入
  → claude logs <id> 查看输出
```

### CI/CD 自动化

```
CI 触发(PR / Push / Issue)
  → claude -p "query"(非交互)
  → 工具执行(权限预配置)
  → 结构化输出(--output-format json)
  → PR 评论 / 状态检查
```

## 关键设计原则

| 原则 | 实践含义 |
|------|----------|
| **多表面统一** | 一个引擎服务所有入口,配置跨表面通用 |
| **渐进式加载** | 技能/规则按需加载,最小化上下文消耗 |
| **权限分层** | 模式 + 规则 + Hooks 三层控制,精细到每个工具调用 |
| **可组合性** | CLI 遵循 Unix 哲学,支持管道、脚本、CI |
| **上下文隔离** | 子代理独立上下文,不污染主对话 |
| **热重载** | 设置文件变更自动重载,无需重启 |
| **可中断** | Esc 中断当前操作,`Shift+Tab` 切换模式 |
