## 系统要求

| 维度 | 要求 |
|------|------|
| **操作系统** | macOS 13.0+、Windows 10 1809+ / Windows Server 2019+、Ubuntu 20.04+、Debian 10+、Alpine 3.19+ |
| **硬件** | 4 GB+ 内存,x64 或 ARM64 处理器 |
| **网络** | 需要互联网连接 |
| **Shell** | Bash、Zsh、PowerShell 或 CMD |
| **地区** | Anthropic 支持的国家/地区 |
| **额外依赖** | ripgrep(通常随 Claude Code 内置) |

## 安装方式

Claude Code 提供多种安装方式,按平台和偏好选择:

| 方式 | 适用平台 | 特点 | 自动更新 |
|------|----------|------|----------|
| **原生安装**(推荐) | macOS / Linux / WSL / Windows | 官方脚本安装,后台自动更新 | 是 |
| **Homebrew** | macOS | `claude-code`(stable)或 `claude-code@latest` | 否(手动 `brew upgrade`) |
| **WinGet** | Windows | `winget install Anthropic.ClaudeCode` | 否(手动 `winget upgrade`) |
| **Linux 包管理器** | Debian / Fedora / RHEL / Alpine | apt / dnf / apk,签名仓库 | 否(随系统升级) |
| **npm**(已弃用) | 全平台 | `npm install -g @anthropic-ai/claude-code` | 受限 |

> **类比**:原生安装像"自动更新App",装完就不管了;Homebrew/WinGet 像"应用商店手动更新";Linux 包管理器像"系统级软件源";npm 已是遗留路径,不再推荐。

### 原生安装命令

| 平台 | 命令 |
|------|------|
| macOS / Linux / WSL | `curl -fsSL https://claude.ai/install.sh \| bash` |
| Windows PowerShell | `irm https://claude.ai/install.ps1 \| iex` |
| Windows CMD | `curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd` |

> 注意:PowerShell 用 `irm`(不是 CMD),CMD 用 `curl`(不是 PowerShell)。提示符带 `PS` 的是 PowerShell。

## Windows 环境配置

Windows 支持两种运行模式,按项目位置和所需功能选择:

| 选项 | 前置要求 | 沙箱支持 | 适用场景 |
|------|----------|----------|----------|
| **原生 Windows** | 无;Git for Windows 可选 | 不支持 | Windows 原生项目和工具链 |
| **WSL 2** | WSL 2 已启用 | 支持 | Linux 工具链或沙箱命令执行 |
| **WSL 1** | WSL 1 已启用 | 不支持 | WSL 2 不可用时 |

**Git for Windows** 的作用:

- 安装后:Claude Code 使用 Git Bash 作为 Bash 工具
- 未安装:使用 PowerShell 工具执行命令
- 路径设置:在 `settings.json` 中配置 `CLAUDE_CODE_GIT_BASH_PATH`

> **类比**:原生 Windows 是"在 Windows 上直接跑",WSL 是"在 Linux 子系统里跑"——后者获得完整 Linux 环境和沙箱能力。

## 自动更新

原生安装支持后台自动更新:

| 机制 | 说明 |
|------|------|
| **检查时机** | 启动时检查,运行中周期性检查 |
| **安装方式** | 后台下载安装,下次启动 Claude Code 时生效 |
| **查看结果** | `claude doctor` 显示最近一次更新尝试结果 |

### 发布渠道

| 渠道 | 说明 |
|------|------|
| `latest`(默认) | 新功能发布即推送 |
| `stable` | 落后约一周,跳过有重大回归的版本 |

通过 `/config` → Auto-update channel 设置,或写入 `settings.json`:

```json
{
  "autoUpdatesChannel": "stable"
}
```

> Homebrew 不使用此设置,而是通过 cask 名称选择:`claude-code`(stable)vs `claude-code@latest`。

### 锁定最低版本

`minimumVersion` 设置建立版本下限,防止自动更新降级:

```json
{
  "autoUpdatesChannel": "stable",
  "minimumVersion": "2.1.100"
}
```

### 禁用自动更新

在 `settings.json` 的 `env` 键中设置:

```json
{
  "env": {
    "DISABLE_AUTOUPDATER": "1"
  }
}
```

> `DISABLE_AUTOUPDATER` 仅停止后台检查;`claude update` 和 `claude install` 仍可用。要阻止所有更新路径(含手动),设置 `DISABLE_UPDATES`。

## 验证安装

```bash
claude --version   # 确认版本
claude doctor      # 详细诊断安装与配置
```

> `claude doctor` 是排障第一站——检查环境、配置、更新状态并给出修复建议。

## Alpine Linux 特殊要求

Alpine 和 musl/uClibc 发行版需要额外安装依赖:

```bash
apk add libgcc libstdc++ ripgrep
```

然后在 `settings.json` 中禁用内置 ripgrep:

```json
{
  "env": {
    "USE_BUILTIN_RIPGREP": "0"
  }
}
```

## 二进制完整性

所有 Linux 包管理器仓库均使用 **Claude Code release signing key** 签名:

| 项目 | 值 |
|------|-----|
| **密钥指纹** | `31DD DE24 DDFA B679 F42D 7BD2 BAA9 29FF 1A7E CACE` |
| **验证方式** | `gpg --show-keys` (apt) 或 dnf 首次安装时确认指纹 |

> 部署前务必验证指纹匹配,确保二进制未被篡改。
