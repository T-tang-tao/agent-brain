## 内置工具一览

Claude Code 内置一组工具,工具名即权限规则中使用的精确字符串:

| 工具 | 说明 | 需要权限 |
|------|------|----------|
| `Agent` | 生成子代理处理任务,独立上下文窗口 | 否 |
| `Bash` | 执行 Shell 命令 | 是 |
| `Edit` | 对文件做定向编辑(old_str/new_str 搜索替换) | 是 |
| `Glob` | 按模式匹配查找文件 | 否 |
| `Grep` | 搜索文件内容 | 否 |
| `LSP` | 语言服务器代码智能(定义跳转、引用查找、类型错误) | 否 |
| `Monitor` | 后台运行命令并逐行反馈输出 | 是 |
| `NotebookEdit` | 修改 Jupyter notebook 单元格 | 是 |
| `PowerShell` | 原生执行 PowerShell 命令 | 是 |
| `Read` | 读取文件内容 | 否 |
| `WebFetch` | 获取指定 URL 内容 | 是 |
| `WebSearch` | 执行网页搜索 | 是 |
| `Write` | 创建或覆盖文件 | 是 |
| `Skill` | 在主对话中执行技能 | 是 |
| `TaskCreate` | 创建新任务 | 否 |
| `TaskList` | 列出所有任务及状态 | 否 |
| `TaskUpdate` | 更新任务状态/依赖/详情 | 否 |
| `TaskGet` | 获取单个任务详情 | 否 |
| `AskUserQuestion` | 多选题收集需求或澄清 | 否 |
| `EnterPlanMode` | 切换到计划模式设计方案 | 否 |
| `ExitPlanMode` | 呈现计划并退出计划模式 | 是 |
| `CronCreate` | 安排会话内定时任务 | 否 |
| `CronDelete` | 按 ID 取消定时任务 | 否 |
| `CronList` | 列出会话内所有定时任务 | 否 |
| `ToolSearch` | 搜索并加载延迟工具 | 否 |

> **类比**:工具集像"Agent 的双手"——Bash/PowerShell 是"执行命令的手",Read/Edit/Write 是"操作文件的手",Grep/Glob 是"搜索的眼",Agent 是"分身术",Skill 是"按需翻阅的SOP手册"。

## 权限规则语法

权限规则统一使用 `ToolName(specifier)` 格式,specifier 因工具而异,部分工具共享同一格式:

| 规则格式 | 作用于 | 说明 |
|----------|--------|------|
| `Bash(npm run *)` | Bash、Monitor | 命令模式匹配 |
| `Read(~/secrets/**)` | Read、Grep、Glob、LSP | 路径模式匹配 |
| `Edit(/src/**)` | Edit、Write、NotebookEdit | 路径模式匹配 |
| `Skill(deploy *)` | Skill | 技能名称匹配 |
| `Agent(Explore)` | Agent | 子代理类型匹配 |
| `WebFetch(domain:example.com)` | WebFetch | 域名匹配 |
| `WebSearch` | WebSearch | 无 specifier,整体允许/拒绝 |

> `Edit(...)` 允许规则同时授予同路径读访问,无需额外配 `Read(...)`。未列出的工具(如 `ExitPlanMode`)仅接受裸工具名,无 specifier。

### 禁用工具

将工具名加入 `permissions` 的 `deny` 数组即可完全禁用:

```json
{
  "permissions": {
    "deny": ["WebFetch"]
  }
}
```

## Bash 工具行为

Bash 工具每条命令在**独立进程**中执行,持久化行为如下:

| 行为 | 说明 |
|------|------|
| **工作目录** | `cd` 在主会话内延续到后续命令(需在项目目录内);子代理会话不延续 |
| **环境变量** | 不持久;一条命令中的 `export` 不会传到下一条 |
| **别名/函数** | 会话启动时加载 `~/.zshrc`、`~/.bashrc` 或 `~/.profile` 中的别名和函数 |
| **限制** | 每条命令有超时和输出上限 |

> 若 `cd` 跳到项目目录外,Claude Code 重置到项目目录并提示。设置 `CLAUDE_BASH_MAINTAIN_PROJECT_WORKING_DIR=1` 可禁用目录延续。

## Edit 工具行为

Edit 工具做**定向编辑**,使用搜索替换模式:

- 提供 `old_str`(搜索段)和 `new_str`(替换段)
- 精确匹配文件中连续的行块,替换为新内容
- SEARCH 和 REPLACE 必须不同,且足够唯一以避免歧义

> **类比**:Edit 像"外科手术刀"——不是整文件重写,而是精准定位一段替换,对周围内容零伤害。

## PowerShell 工具

PowerShell 工具是 Windows 原生命令执行工具:

| 项目 | 说明 |
|------|------|
| **启用方式** | 设置 `CLAUDE_CODE_USE_POWERSHELL_TOOL=1` |
| **关系** | 与 Bash 工具并存(Git for Windows 安装时逐步推出) |
| **关闭** | 设置 `CLAUDE_CODE_USE_POWERSHELL_TOOL=0` |

未安装 Git for Windows 时,Claude Code 默认使用 PowerShell 工具执行命令。

## Agent 工具行为

Agent 工具生成一个在**独立上下文窗口**中运行的子代理:

| 行为 | 说明 |
|------|------|
| **返回值** | 子代理自主完成任务后返回单一文本结果给父对话 |
| **中间过程** | 父对话看不到子代理的中间工具调用和输出,只看最终结果 |
| **权限** | 生成子代理本身不触发权限提示;子代理运行时按各自权限规则检查 |
| **工具继承** | 未设 `tools`/`disallowedTools` 时继承父代理全部工具 |
| **后台运行** | v2.1.198+ 默认后台运行;需要结果时前台运行 |

> **类比**:Agent 像"派助理去独立干活"——你交代任务,他独立完成,回来只给你最终报告,过程你不用操心。

## 扩展工具的方式

| 扩展方式 | 说明 |
|----------|------|
| **MCP Server** | 连接 MCP 服务器添加自定义工具 |
| **Skill** | 编写技能通过现有 `Skill` 工具运行,不新增工具条目 |
| **Plugin** | 安装插件打包工具、钩子、MCP 等 |

> 添加技能不会新增模型工具条目——技能通过 `Skill` 工具执行,保持核心工具集精简。
