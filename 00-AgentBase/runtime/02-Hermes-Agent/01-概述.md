## 这是什么

Hermes Agent 是 Nous Research 开发的个人 AI Agent。它的核心特征是**自我进化**——不是被动执行指令,而是在使用过程中持续学习:

- 完成复杂任务后**自动创建技能**(skill)
- 技能在使用中**自我改进**
- 主动**持久化知识**到记忆
- **搜索自己的历史对话**(FTS5 全文检索 + LLM 摘要)
- 跨会话**建立用户画像**(Honcho 辩证建模)

## 类比

大多数 Agent 像"临时工"——每次任务从头开始,用完就忘。Hermes 像"长期合伙人"——记住你、积累技能、越用越懂你。

## 核心特性

| 特性 | 说明 |
|------|------|
| **闭环学习** | Agent 管理的记忆 + 周期性提醒。任务后自主创建技能,技能使用中自我改进。FTS5 会话搜索 + LLM 摘要实现跨会话回忆 |
| **完整终端界面** | TUI 支持多行编辑、斜杠命令自动补全、对话历史、中断重定向、流式工具输出 |
| **多平台在线** | Telegram、Discord、Slack、WhatsApp、Signal、Matrix、Email、SMS、飞书、企微、钉钉等 20+ 平台——一个 gateway 进程全搞定。支持语音转文字、跨平台对话连续性 |
| **定时自动化** | 内置 cron 调度器,可投递到任何平台。日报、夜备份、周审计——全部自然语言,无人值守 |
| **委托与并行** | 生成隔离子 Agent 处理并行工作流。可用 Python 脚本通过 RPC 调用工具,把多步管道压缩成零上下文成本的单轮 |
| **随处运行** | 六种终端后端:local、Docker、SSH、Singularity、Modal、Daytona。后两者支持 serverless 持久化——空闲时休眠,按需唤醒 |
| **MCP 支持** | 连接任意 MCP server 扩展工具能力,支持 stdio 和 HTTP 两种传输,内置目录一键安装 |
| **研究就绪** | 批量轨迹生成,轨迹压缩用于训练下一代工具调用模型 |

## 架构关键设计

Hermes 的设计有两条贯穿所有决策的核心原则:

### 原则一:对话级 Prompt 缓存是神圣的

长对话每轮都复用缓存的 prefix。任何修改过去上下文、切换工具集、或重建 system prompt 的行为都会使缓存失效,成倍增加成本。Hermes 不做这些(唯一例外是上下文压缩)。

> **类比**:像 Git 的 commit hash——一旦写入就不可变,改了就全部失效。

### 原则二:核心是窄腰,能力在边缘

每个模型工具都会随每次 API 调用发送,所以新增**核心工具**的门槛很高。大多数新能力应该通过以下"足迹阶梯"加入:

```
足迹从小到大:
1. 扩展现有代码          ← 零新增表面
2. CLI 命令 + Skill      ← 零模型工具足迹(默认选择)
3. 服务门控工具(check_fn) ← 配置后才出现
4. Plugin                ← 第三方/小众/用户特定
5. MCP Server(目录中)    ← 零核心 schema 足迹,任何 MCP host 可复用
6. 新核心工具            ← 最后手段,仅限基础且通用能力
```

> **类比**:核心工具像"操作系统内核调用"——不能随便加;Skill/Plugin/MCP 像"应用程序"——可以自由扩展。

## 设计原则总览

| 原则 | 含义 |
|------|------|
| **Prompt 稳定性** | 系统提示在对话中不变,不做缓存失效操作(用户显式 `/model` 除外) |
| **可观测执行** | 每次工具调用对用户可见,CLI 有 spinner、Gateway 有进度消息 |
| **可中断** | API 调用和工具执行可被用户输入或信号中途取消 |
| **平台无关核心** | 一个 AIAgent 类服务 CLI、Gateway、ACP、Batch、API Server |
| **松耦合** | 可选子系统(MCP、插件、记忆 provider、RL 环境)用注册表模式,非硬依赖 |
| **Profile 隔离** | 每个 profile 有独立的 HERMES_HOME、配置、记忆、会话、Gateway PID |

## 项目结构要点

```
hermes-agent/
├── run_agent.py        # AIAgent 类—核心对话循环
├── model_tools.py      # 工具编排,discover_builtin_tools(),handle_function_call()
├── toolsets.py         # 工具集定义
├── cli.py              # HermesCLI 类—交互式 CLI 编排器
├── hermes_state.py     # SessionDB—SQLite 会话存储(FTS5 搜索)
├── agent/              # Agent 内部(provider 适配器、记忆、缓存、压缩)
├── tools/              # 工具实现—通过 registry.py 自动发现
│   └── environments/   # 终端后端(local/docker/ssh/modal/daytona/singularity)
├── gateway/            # 消息网关
│   └── platforms/      # 平台适配器(20+ 平台)
├── plugins/            # 插件系统
├── skills/             # 内置技能
├── optional-skills/    # 重型/小众技能(默认不启用)
├── cron/               # 调度器
└── ui-tui/             # Ink(React)终端 UI
```

## 核心命令

| 命令 | 作用 |
|------|------|
| `hermes` | 启动交互式 CLI 对话 |
| `hermes --tui` | 启动现代 TUI 界面(推荐) |
| `hermes model` | 选择 LLM provider 和模型 |
| `hermes tools` | 配置启用的工具 |
| `hermes gateway` | 启动消息网关(Telegram/Discord 等) |
| `hermes setup` | 运行完整设置向导 |
| `hermes setup --portal` | Nous Portal 一键配置(模型 + 工具网关) |
| `hermes cron` | 管理定时任务 |
| `hermes update` | 更新到最新版本 |
| `hermes doctor` | 诊断问题 |
