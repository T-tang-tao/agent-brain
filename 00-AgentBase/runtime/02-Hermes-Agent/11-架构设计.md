## 系统概览

```
入口点
  CLI (cli.py) | Gateway (gateway/run.py) | ACP | Batch Runner | API Server | Python Library
                    ↓
  AIAgent (run_agent.py)
    ├── Prompt Builder → Compression & Caching
    ├── Provider Resolution → 3 API Modes (chat_completions / codex_responses / anthropic)
    └── Tool Dispatch → Tool Registry (70+ tools, 28 toolsets)
                    ↓
  Session Storage (SQLite + FTS5) | Tool Backends (Terminal/Browser/Web/MCP/File)
```

> 一个 `AIAgent` 类服务所有入口点——这是"平台无关核心"原则的体现。

## 主要子系统

| 子系统 | 核心文件 | 说明 |
|---|---|---|
| Agent Loop | `run_agent.py` | 同步编排引擎：provider 选择、prompt 构建、工具执行、重试、回退、压缩、持久化 |
| Prompt 系统 | `agent/prompt_builder.py` | 从 SOUL.md、MEMORY.md、USER.md、skills、上下文文件组装系统提示 |
| Provider 解析 | `hermes_cli/runtime_provider.py` | 将 (provider, model) 映射到 (api_mode, api_key, base_url)，处理 18+ provider |
| 工具系统 | `tools/registry.py` | 中央工具注册表，70+ 工具，每个工具文件导入时自动注册 |
| 会话持久化 | `hermes_state.py` | SQLite + FTS5 全文搜索，会话血缘追踪，per-platform 隔离 |
| 消息网关 | `gateway/run.py` | 长运行进程，20 平台适配器，统一会话路由，用户授权 |
| 插件系统 | `hermes_cli/plugins.py` | 三种发现源：~/.hermes/plugins/、.hermes/plugins/、pip entry points |
| Cron | `cron/scheduler.py` | 一等 Agent 任务（非 shell 任务），JSON 存储，支持多技能 |
| ACP 集成 | `acp_adapter/` | 通过 stdio/JSON-RPC 暴露为编辑器原生 Agent（VS Code/Zed/JetBrains） |
| 轨迹 | `agent/trajectory.py` | 生成 ShareGPT 格式轨迹用于训练数据 |

> 类比：AIAgent 是"大脑"，Prompt 系统是"语言中枢"，工具系统是"双手"，会话持久化是"记忆"，网关是"耳朵和嘴"。

## 数据流

### CLI 会话

```
用户输入
  → HermesCLI.process_input()
  → AIAgent.run_conversation()
  → prompt_builder 构建系统提示
  → runtime_provider 解析 provider
  → API 调用
  → 工具调用循环
  → 最终响应
  → 显示
  → 保存到 SessionDB
```

### Gateway 消息

```
平台事件
  → Adapter.on_message()
  → GatewayRunner._handle_message()
  → 授权用户
  → 解析会话 key
  → 创建 AIAgent
  → run_conversation()
  → 通过 adapter 投递响应
```

### Cron 任务

```
调度器 tick
  → 从 jobs.json 加载到期任务
  → 创建新 AIAgent（无历史）
  → 注入技能
  → 运行 prompt
  → 投递响应到目标平台
  → 更新任务状态
```

> 三条数据流的共同终点都是 `AIAgent.run_conversation()`——核心编排逻辑只有一个，入口点的差异在进入核心前被抹平。

## 设计原则

| 原则 | 实践含义 |
|---|---|
| Prompt 稳定性 | 系统提示在对话中不变，不做缓存失效操作 |
| 可观测执行 | 每次工具调用对用户可见 |
| 可中断 | API 调用和工具执行可中途取消 |
| 平台无关核心 | 一个 AIAgent 类服务所有入口点 |
| 松耦合 | 可选子系统用注册表模式，非硬依赖 |
| Profile 隔离 | 每个 profile 独立的 HERMES_HOME、配置、记忆、会话 |

## 文件依赖链

```
tools/registry.py (无依赖 — 被所有工具文件导入)
    ↑
tools/*.py (每个在导入时调用 registry.register())
    ↑
model_tools.py (导入 registry + 触发工具发现)
    ↑
run_agent.py, cli.py, batch_runner.py, environments/
```

工具注册在**导入时**发生，在任何 Agent 实例创建之前。任何带顶层 `registry.register()` 调用的 `tools/*.py` 文件自动被发现。

> 这是"松耦合"原则的具体实现：新增工具只需创建文件并调用注册函数，无需修改核心代码。
