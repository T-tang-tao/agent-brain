## 快速安装

Hermes 提供多种安装方式,按平台和偏好选择:

| 方式 | 适用平台 | 命令 / 操作 |
|------|----------|-------------|
| **桌面安装器** | macOS / Windows(推荐) | 从官网下载 Hermes Desktop installer,双击安装 |
| **命令行安装** | Linux / macOS / WSL2 / Termux | `curl -fsSL https://hermes-agent.nousresearch.com/install.sh \| bash` |
| **Windows 原生** | Windows(PowerShell) | `iex (irm https://hermes-agent.nousresearch.com/install.ps1)` |
| **pip 安装** | 全平台(Python 环境) | `pip install hermes-agent` + `hermes postinstall` |
| **桌面应用安装后命令行** | 已装桌面版的机器 | `hermes desktop` |

> **类比**:桌面安装器像"应用商店一键装 App",命令行脚本像"包管理器自动配齐一切",pip 安装像"手动组装零件"。

## 安装器自动处理

安装器会**自动安装所有依赖**,无需手动准备:

- uv(Python 包管理器)
- Python 3.11
- Node.js v22
- ripgrep(代码搜索)
- ffmpeg(音视频处理)

**无需管理员权限**。唯一前置要求:

| 平台 | 前置要求 |
|------|----------|
| 全平台 | Git |
| Linux | 额外需要 `curl` 和 `xz-utils` |

## 安装布局

Hermes 支持两种安装模式,文件位置不同:

| 安装方式 | 代码位置 | hermes 命令 | 数据目录 |
|----------|----------|-------------|----------|
| **Per-user**(默认) | `~/.hermes/hermes-agent/` | `~/.local/bin/hermes` | `~/.hermes/` |
| **Root 模式** | `/usr/local/lib/hermes-agent/` | `/usr/local/bin/hermes` | `/root/.hermes/` |

> Per-user 模式不污染系统目录,推荐个人使用;Root 模式适合多用户共享的服务器场景。

## 安装后

```bash
source ~/.bashrc  # 重载 shell(让 hermes 命令生效)
hermes            # 开始对话
```

首次运行后的后续配置命令:

| 命令 | 作用 |
|------|------|
| `hermes model` | 选择 LLM provider 和模型 |
| `hermes tools` | 配置启用的工具 |
| `hermes gateway setup` | 配置消息网关平台 |
| `hermes config set KEY VAL` | 设置单个配置项 |
| `hermes setup` | 运行完整设置向导 |

## Nous Portal 一键配置

```bash
hermes setup --portal
```

一次 OAuth 授权即可覆盖:

- **300+ 模型**:无需逐个配置 API key
- **工具网关**:web search、image generation、TTS、browser

> 这是新用户最快的上手路径——一条命令搞定模型 + 工具。

## 非 Sudo / 系统服务用户安装

支持**无 sudo 环境**安装。注意 Playwright 的 `--with-deps` 需要 root,安装器会检测并**优雅降级**。

推荐分步部署(管理员 + 服务用户协作):

```bash
# 第 1 步:admin 先装 Chromium 依赖(需要 root)
sudo npx playwright install-deps chromium

# 第 2 步:service user 跑安装器(无需 sudo)
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
```

如完全不需要浏览器功能,可用 `--skip-browser` 跳过 Playwright 安装:

```bash
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash -s -- --skip-browser
```

## 模型 Provider 要求

| 要求 | 说明 |
|------|------|
| **最低上下文** | 64K token |
| **支持 Provider 数** | 30+ |

支持的 provider 包括:

- Nous Portal(推荐,一键配置)
- OpenAI、Anthropic、OpenRouter
- Google、AWS Bedrock、Azure Foundry
- DeepSeek、Qwen、Kimi
- 任意 OpenAI 兼容端点

## 故障排查

| 常见问题 | 解决方案 |
|----------|----------|
| `hermes: command not found` | 重载 shell(`source ~/.bashrc`)或检查 `~/.local/bin` 是否在 PATH 中 |
| `API key not set` | 运行 `hermes model` 配置 provider 和密钥 |
| Missing config after update | 运行 `hermes config check` 检查,必要时 `hermes config migrate` 迁移 |
| 其他未知问题 | 运行 `hermes doctor` 进行全面诊断 |

> `hermes doctor` 是排障第一站——它会检查环境、依赖、配置、网络连通性并给出修复建议。

## 安装方式自动检测

Hermes 会自动检测自身的安装方式(pip / git / Homebrew / NixOS),`hermes update` 会打印对应的更新命令:

| 安装方式 | 更新命令 |
|----------|----------|
| pip | `pip install --upgrade hermes-agent` |
| git | `git pull` |
| Homebrew | `brew upgrade hermes` |
| NixOS | `nix flake update` |

无需记住自己当初怎么装的——Hermes 替你记着。
