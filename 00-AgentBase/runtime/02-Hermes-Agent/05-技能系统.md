## 这是什么

技能(Skill)是 Agent **按需加载的知识文档**。它不是代码,而是指导 Agent 如何完成特定任务的"操作手册"。

核心设计遵循**渐进式加载**(progressive disclosure)模式,以最小化 token 消耗——Agent 先看到技能目录,只在实际需要时才加载完整内容。

关键特征:

- 兼容 [agentskills.io](https://agentskills.io) 开放标准
- 所有技能存储在 `~/.hermes/skills/`
- Agent 可自主创建和改进技能(闭环学习)
- 支持社区共享与外部目录集成

> **类比**:技能像"按需查阅的 SOP 手册"——书架上摆着目录(只看标题),需要时才抽出来翻完整内容,用完放回。

## 渐进式加载

技能加载分三个层级,逐级深入:

```
Level 0: skills_list()
    → [{name, description, category}]  (~3k tokens, 仅摘要)
    Agent 从目录中判断哪些技能可能相关

Level 1: skill_view(name)
    → 完整内容 + 元数据
    加载某个技能的完整 SKILL.md

Level 2: skill_view(name, path)
    → 特定引用文件
    仅加载技能目录中的某个 references 文件
```

**为什么这样设计**:70+ 技能如果全部展开会消耗大量 token。Level 0 只传摘要(~3k tokens),Agent 据此决定深入哪个技能,避免无关技能污染上下文。

## 使用技能

每个已安装技能**自动成为斜杠命令**,无需额外注册:

```
/gif-search funny cats
/axolotl help me fine-tune Llama 3
/github-pr-workflow create a PR
```

### 多技能叠加

支持在一次调用中叠加多个技能(最多 5 个),后一个技能的输出作为前一个的输入:

```
/github-pr-workflow /test-driven-development fix issue #123
```

> **类比**:像 Unix 管道——`skill A | skill B`,前者的输出喂给后者,组合出复杂工作流。

## /learn 从来源学习

`/learn` 是 Hermes 闭环学习的关键入口——将已有材料转化为可复用技能,无需手写 SKILL.md:

```
# 从本地代码库学习
/learn the REST client in ~/projects/acme-sdk

# 从在线文档学习
/learn https://docs.example.com/api/quickstart

# 从刚完成的操作学习(回溯当前会话)
/learn how I just deployed the staging server
```

Agent 分析材料后自动生成 SKILL.md,存入 `~/.hermes/skills/`,后续可按斜杠命令调用。

## SKILL.md 格式

技能的核心是 `SKILL.md` 文件,由 YAML front matter 和 Markdown 正文组成:

```yaml
---
name: my-skill
description: 简短描述
version: 1.0.0
platforms: [macos, linux]  # 可选:限制操作系统
metadata:
  hermes:
    tags: [python, automation]
    category: devops
    fallback_for_toolsets: [web]  # 条件激活
    requires_toolsets: [terminal] # 条件激活
---
# 技能标题
## When to Use      # 何时使用
## Procedure        # 操作步骤
## Pitfalls         # 常见陷阱
## Verification     # 验证方法
```

正文采用固定结构,确保 Agent 能快速定位所需信息:

| 章节 | 作用 |
|---|---|
| When to Use | 判断当前任务是否适用此技能 |
| Procedure | 具体操作步骤 |
| Pitfalls | 常见错误与避坑指南 |
| Verification | 如何确认任务完成 |

## 平台特定技能

`platforms` 字段限制技能在特定操作系统上加载:

```yaml
platforms: [macos, linux]  # 仅在 macOS 和 Linux 上可用
```

支持的平台值:`macos`、`linux`、`windows`。未声明 `platforms` 的技能在所有平台可用。

## 条件激活

技能可根据当前可用工具集/工具条件激活,避免不相关的技能出现在目录中:

| 字段 | 行为 |
|---|---|
| `fallback_for_toolsets` | 当列出的工具集**不可用**时显示此技能 |
| `requires_toolsets` | 当列出的工具集**可用**时显示此技能 |
| `fallback_for_tools` | 同上,针对单个工具 |
| `requires_tools` | 同上,针对单个工具 |

**示例**:一个"手动 web 搜索"技能,在 `web` 工具集不可用时作为 fallback 出现:

```yaml
metadata:
  hermes:
    fallback_for_toolsets: [web]
```

## 安全加载设置

技能可声明所需的环境变量。当声明的变量缺失时,Hermes 会提示用户进行安全设置,而非静默失败。

```yaml
metadata:
  hermes:
    required_env: [API_KEY, DATABASE_URL]
```

**重要**:声明的环境变量会**自动传递到** `execute_code` 和 `terminal` 沙箱中,确保技能内的代码执行能访问所需凭证,同时避免将全部环境变量无差别暴露。

## 技能目录结构

所有技能按分类组织在 `~/.hermes/skills/` 下:

```
~/.hermes/skills/
├── mlops/
│   ├── axolotl/
│   │   ├── SKILL.md          # 技能定义
│   │   ├── references/       # 引用文件(Level 2 加载)
│   │   └── scripts/          # 辅助脚本
│   └── vllm/
├── devops/
│   └── deploy-k8s/
└── .hub/                     # 社区共享技能缓存
```

每个技能是一个目录,核心文件为 `SKILL.md`,可附带 `references/`(详细文档)和 `scripts/`(辅助脚本)。

## 外部技能目录

`config.yaml` 中配置 `external_dirs`,将项目级或团队级技能目录纳入 Hermes:

```yaml
skills:
  external_dirs:
    - ~/projects/acme-team/skills
    - ${SKILLS_REPO}/shared
```

特性:

- 支持 `~` 主目录展开和 `${VAR}` 环境变量替换
- **本地优先**:同名技能本地版本覆盖外部版本
- 外部技能与本地技能**完全集成**,在 `skills_list()` 中统一展示

## 技能包(Bundles)

将多个技能组合在一个斜杠命令下,形成工作流模板:

```bash
hermes bundles create backend-dev \
  --skill github-code-review \
  --skill test-driven-development \
  -d "Backend feature work"
```

创建后即可一次性激活多个技能:

```
/backend-dev refactor the auth middleware
```

> **类比**:像"快捷方式"——把一组常用技能打包,一次调用全部就位,省去逐个叠加的麻烦。

## 空白启动

不想要内置种子技能时,可以完全空白启动:

```bash
# 安装时不植入种子技能
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash -s -- --no-skills

# 停止未来更新中的种子技能植入
hermes skills opt-out

# 同时删除未修改的内置技能(已修改的保留)
hermes skills opt-out --remove
```

适用场景:企业环境需严格控制技能来源,或用户希望完全从零自建技能库。

## 技能 vs 记忆

技能和记忆都是 Hermes 的知识承载机制,但定位不同:

| 维度 | 技能(Skill) | 记忆(Memory) |
|---|---|---|
| 形态 | 结构化文档(SKILL.md) | 自由文本片段 |
| 触发 | 斜杠命令主动调用 | Agent 自动检索 |
| 粒度 | 完整操作流程 | 事实、偏好、上下文 |
| 加载 | 渐进式三级 | 按相关性召回 |
| 创建 | `/learn` 或手写 | Agent 自动持久化 |

详见 [Hermes 记忆系统](./06-记忆系统.md)。
