## 配置目录结构

所有配置集中在 `~/.hermes/` 目录下:

```
~/.hermes/
├── config.yaml     # 设置(模型、终端、TTS、压缩等)
├── .env            # API 密钥和密钥
├── auth.json       # OAuth 凭证
├── SOUL.md         # Agent 身份
├── memories/       # 持久记忆(MEMORY.md, USER.md)
├── skills/         # Agent 创建的技能
├── cron/           # 定时任务
├── sessions/       # Gateway 会话
└── logs/           # 日志(密钥自动脱敏)
```

> **类比**:`~/.hermes/` 像 Agent 的"家目录"——身份(SOUL.md)、记忆(memories/)、技能(skills/)、日程(cron/)、日志(logs/)各居其位。

## 配置管理命令

| 命令 | 作用 |
|------|------|
| `hermes config` | 查看当前配置概览 |
| `hermes config edit` | 用编辑器打开 config.yaml |
| `hermes config set KEY VAL` | 设置单个配置项 |
| `hermes config check` | 检查配置完整性和一致性 |
| `hermes config migrate` | 迁移旧版配置到新格式 |

`hermes config set` 会**自动路由**:API key 类配置写入 `.env`,其他配置写入 `config.yaml`。

## 配置优先级

配置按以下优先级从高到低生效:

```
1. CLI 参数              ← 最高优先级
2. ~/.hermes/config.yaml
3. ~/.hermes/.env
4. 内置默认值             ← 最低优先级
```

**存放规则**:

| 配置类型 | 存放位置 |
|----------|----------|
| 密钥(API key、bot token、密码) | `.env` |
| 其他设置(模型、终端、TTS 等) | `config.yaml` |

> 当 `config.yaml` 和 `.env` 同时设置同一项时,`config.yaml` 胜出。

## 环境变量替换

`config.yaml` 中支持 `${VAR_NAME}` 语法引用环境变量:

```yaml
# 示例:从环境变量读取模型名
model: ${HERMES_MODEL}
```

规则:

- 只支持 `${VAR}` 语法,**不支持** `$VAR`
- 未设置的变量**保留原样**(不报错,字符串原样保留)

## 终端后端配置

Hermes 支持六种终端后端,决定命令执行的隔离程度:

| 后端 | 命令执行位置 | 隔离级别 | 适用场景 |
|------|-------------|----------|----------|
| **local** | 本机 | 无隔离 | 开发、个人使用 |
| **docker** | 单一持久容器 | 完整隔离 | 安全沙箱、CI/CD |
| **ssh** | 远程服务器 | 网络边界 | 远程开发 |
| **modal** | Modal 云沙箱 | 完整隔离 | 弹性云计算 |
| **daytona** | Daytona 工作区 | 完整隔离 | 托管云开发环境 |
| **singularity** | Singularity 容器 | 命名空间 | HPC 集群 |

> **类比**:local 是"在自己电脑上直接跑",docker 是"在隔离沙箱里跑",ssh 是"在远程机器上跑",modal/daytona 是"在云上按需起停",singularity 是"HPC 集群的轻量隔离"。

## home_mode

控制 HOME 目录在容器环境中的映射方式:

| 模式 | 主机 HOME | 容器 HOME | 说明 |
|------|-----------|-----------|------|
| **auto**(推荐默认) | 真实 HOME | `{HERMES_HOME}/home` | 主机保持真实,容器用隔离目录 |
| **real** | 真实 HOME | 真实 HOME | 强制使用真实 HOME |
| **profile** | per-profile | per-profile | 严格 per-profile 隔离 |

## Docker 后端

Docker 后端使用**单一持久容器**,跨会话共享状态:

- 工作目录变更**保留**
- 安装的包**保留**
- 创建的文件**保留**

所有状态跨工具调用持久存在。

### 关键配置项

| 配置项 | 作用 |
|--------|------|
| `docker_image` | 容器基础镜像 |
| `docker_forward_env` | 从主机转发到容器的环境变量列表 |
| `docker_env` | 注入容器的环境变量 |
| `docker_volumes` | 挂载的卷映射 |
| `docker_extra_args` | 附加 docker run 参数 |
| `container_cpu` | CPU 限制 |
| `container_memory` | 内存限制 |
| `container_disk` | 磁盘限制 |
| `container_persistent` | 是否持久化容器 |
| `docker_persist_across_processes` | 是否跨 Hermes 进程重用容器 |

### 容器生命周期

容器通过标签标记和管理:

- **标签**:`hermes-agent=1`、`hermes-task-id`、`hermes-profile`
- **默认持久**:跨 Hermes 进程重用,避免重复创建开销
- **清理触发**:
  - `persist_across_processes=false` 时进程退出即清理
  - idle reaper:清理长时间空闲的容器
  - orphan reaper:清理孤儿容器(异常退出残留)

## 更新行为

```yaml
updates:
  pre_update_backup: false       # 更新前是否备份
  backup_keep: 5                 # 保留的备份数量
  non_interactive_local_changes: stash  # stash | discard
```

| 字段 | 作用 |
|------|------|
| `pre_update_backup` | 更新前是否自动备份当前版本 |
| `backup_keep` | 保留多少个历史备份 |
| `non_interactive_local_changes` | 非交互模式下遇到本地修改时的策略:`stash`(暂存)或 `discard`(丢弃) |

## Provider 超时配置

可针对 provider 和具体模型分别配置超时:

```yaml
providers:
  <id>:
    request_timeout_seconds: 120      # provider 级别默认超时
    models:
      <model>:
        timeout_seconds: 300          # 模型级别覆盖超时
```

模型级配置优先于 provider 级配置。
