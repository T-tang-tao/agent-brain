# 概念全景辨析

> 基础概念解释已迁移至 [00-human/02-核心术语速览-人类版](./00-human/02-核心术语速览-人类版.md)。
>
> 本文件保留项目特定的概念边界和目录映射，供 Agent 快速定位资产。

## 概念到目录映射

| 概念 | 解决的问题 | 放在哪里 |
|------|----------|----------|
| Tool | Agent 能做什么动作 | runtime / MCP / custom tool |
| MCP | 如何连接外部工具和资源 | MCP server / MCP config |
| Skill | 某类任务应该怎么做 | `01-Skills/` |
| Plugin | 如何扩展宿主系统 | `02-Plugins/` |
| Prompt | 当前如何行为 | `03-Prompts/` |
| Knowledge Base | 可以查什么资料 | wiki / docs / RAG |
| Memory | 长期记住什么经验 | memory store / memory files |
| RAG | 如何检索相关知识 | search / vector / BM25 / rerank |

## 项目特定边界

这些边界规则用于区分本知识库中不同目录的职责，避免混淆：

| 边界 | 说明 |
|------|------|
| Skill 不是 MCP | Skill 是流程知识（`01-Skills/`），MCP 是连接协议（`04-MCP/`） |
| MCP 不是知识库 | MCP 可以暴露知识资源，但 MCP 本身不是知识库 |
| Plugin 不是 Prompt | Plugin 是扩展包（`02-Plugins/`），Prompt 是指令文本（`03-Prompts/`） |
| Memory 不是 RAG | Memory 是长期记忆，RAG 是检索方式 |
| Tool 不是 Agent | Tool 是 Agent 调用的动作，Agent 是决策和执行循环 |
| 概念解释不是操作指南 | 概念在 `00-AgentBase/`，操作指南在 `01-06/` |

## 系统配合关系

```text
Prompt 定义当前行为
Skill 提供任务流程
Knowledge Base 提供资料
Memory 提供长期经验
MCP 连接外部系统
Tool 执行动作
Plugin 扩展宿主能力
Runtime 管理循环和权限
Model 负责推理和决策
```
