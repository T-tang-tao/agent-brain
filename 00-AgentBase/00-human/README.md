# 00-human — 人类阅读区

> **这个目录给人类看，不是给 Agent 看的。**
>
> Agent 不需要读这里的内容——这些是人类的**基础认知**，大模型天生就懂。

## 为什么有这个目录

知识库里的内容有两类读者：

| 读者 | 需要什么 | 例子 |
|------|----------|------|
| **人类** | 概念解释、类比、学习路径、误解澄清 | "Agent 和模型有什么区别？""什么是 MCP？" |
| **Agent** | 项目结构、规则、约定、可执行指引 | "这个知识库的目录怎么组织""哪些操作需要确认" |

问题是：人类需要的基础知识，AI 天生就懂。Agent 花上下文窗口读"什么是 Agent""Tool 和 Skill 有什么区别"，纯属浪费——它比人类更清楚这些概念。

所以把人类入门内容**单独隔离**到这个目录：

- 人类从这里入门，建立认知后去读其他模块
- Agent 直接跳过这个目录，从 `01-AgentBase总览.md` 和 `03-知识地图.md` 开始

## 目录结构

```
00-human/
├── README.md                        # 本文件
├── 01-Agent是什么-人类入门.md        # Agent 定义、组成、为什么能做事、和模型/Workflow 的区别
├── 02-核心术语速览-人类版.md          # Tool/MCP/Skill/Plugin/Prompt/知识库/Memory/RAG
├── 03-常见误解与澄清.md              # 关于 Agent 的 7 个常见误解
└── 04-人类如何使用这个知识库.md       # 阅读顺序、学习路径、按角色推荐
```

## 怎么读

**第一次接触 Agent？** 按编号顺序读：

1. [01-Agent是什么](./01-Agent是什么-人类入门.md) — 建立基线认知
2. [02-核心术语速览](./02-核心术语速览-人类版.md) — 搞懂关键词
3. [03-常见误解与澄清](./03-常见误解与澄清.md) — 避开认知陷阱
4. [04-如何使用这个知识库](./04-人类如何使用这个知识库.md) — 知道接下来读什么

读完这 4 篇，就可以去 [01-AgentBase总览](../01-AgentBase总览.md) 正式进入知识库了。

## 写作原则

这个目录的文档和知识库其他部分风格不同：

| 维度 | 00-human（人类区） | 其他目录（Agent 区） |
|------|-------------------|---------------------|
| 语态 | 叙述式、讲故事 | 结构化、查表式 |
| 类比 | 丰富、生活化 | 精简、功能导向 |
| 例子 | 贴近日常 | 贴近工程实践 |
| 长度 | 可以长一些 | 控制在 300 行内 |
| 目标 | 让人类"理解" | 让 Agent"检索" |
