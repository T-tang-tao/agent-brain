# 人类如何使用这个知识库

> 这篇文章帮人类找到在这个知识库里的**学习路径**和**使用方式**。

## 知识库的两层结构

整个 `agent/` 目录分两层：

```
agent/
├── 00-AgentBase/        ← 知识层:概念、认知、设计理念
│   ├── 00-human/        ← 你在这里:人类入门(基础认知)
│   ├── model/           ← 模型:Agent/模型/Workflow 的区别
│   ├── runtime/         ← 运行时:Hermes、Claude Code、Codex CLI
│   ├── tools/           ← 工具:Tool、MCP、Custom
│   ├── knowledge/       ← 知识:Wiki、RAG、Memory
│   ├── behavior/        ← 行为:Prompt、Skill、Policy
│   └── safety/          ← 安全:Permission、Boundary、Verification
│
├── 01-Skills/           ← 资产层:可直接使用的技能
├── 02-Plugins/          ← 资产层:插件配置
├── 03-Prompts/          ← 资产层:提示词模板
├── 04-MCP/              ← 资产层:MCP server 配置
├── 05-Boundaries/       ← 资产层:边界规则
└── 06-Migration/        ← 资产层:迁移 playbook
```

知识层帮你**理解**，资产层帮你**执行**。

## 推荐学习路径

### 路径 1：从零开始（完全不懂 Agent）

```
00-human/ 01 → 02 → 03 → 04（你正在读的这 4 篇）
    ↓
01-AgentBase总览.md（知识库入口）
    ↓
03-知识地图.md（看全局）
    ↓
按兴趣进入七大模块
```

### 路径 2：想搭建 Agent 环境（开发者）

```
01-AgentBase总览.md → 03-知识地图.md
    ↓
runtime/01-Agent运行时与CLI.md（理解 Runtime 概念）
    ↓
runtime/02-Hermes-Agent/ 或 03-ClaudeCode/ 或 04-Codex-CLI/（选一个 Runtime）
    ↓
根目录 AGENTS.md（部署、编排与管理指南）
    ↓
01-Skills/ ~ 06-Migration/（获取实践资产）
```

### 路径 3：想理解知识库设计（架构师）

```
knowledge/01-Agent知识库设计.md
    ↓
knowledge/03-LLM-Wiki架构理念.md
    ↓
knowledge/02-如何构建LLM-Wiki.md
    ↓
01-Skills/kb-init/SKILL.md（实际工具:从零初始化知识库）
```

### 路径 4：想了解 Agent 安全（运维/安全）

```
safety/01-Agent边界与限制.md
    ↓
05-Boundaries/（可执行边界规则）
    ↓
各 Runtime 的安全章节(如 Hermes 10-安全机制、Claude Code 10-权限与安全)
```

## 怎么快速找到需要的内容

1. **知道关键词** → 去 `00-目录索引.md`，按模块查找
2. **想看全局** → 去 `03-知识地图.md`
3. **第一次来** → 从本文开始，按推荐路径走
4. **找可执行资产** → 去 `01-Skills/` 到 `06-Migration/`
5. **找概念解释** → 去 `00-AgentBase/` 对应模块

## 人类和 Agent 读的东西有什么不同

| 人类需要 | Agent 需要 |
|----------|-----------|
| 概念解释 + 类比 | 稳定路径 + 结构化数据 |
| 学习顺序 | 明确小节标题 |
| 误解澄清 | 边界说明(什么不能做) |
| 恰当的例子 | 可抽取的表格 |
| 叙述式讲解 | 相关文件链接 |

这就是为什么 `00-human/` 单独存在：人类需要的东西，对 Agent 是噪音。

## 如何参与建设

如果你想给这个知识库贡献内容：

1. **新概念** → 放对应模块子目录，遵循 `00-AgentBase/AGENTS.md` 的写作标准
2. **新实践资产** → 放 `01-06/` 对应目录
3. **入门内容** → 放 `00-human/`，用人类友好的叙述风格
4. **修正/补充** → 先读现有文档，确认不是重复，再修改

关键原则：
- 概念解释 vs 可执行资产要分离
- 人类入门内容放 `00-human/`，不放其他模块
- 每篇文档不超过 300 行
- 发布前过一遍审查清单（见 `00-AgentBase/AGENTS.md` § 9）
