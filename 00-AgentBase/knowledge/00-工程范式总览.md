# AI 编程工程范式总览

> 2026 年主流的四种 AI 编程工程范式,按抽象层级递增排列。每一层都包裹前一层,不替代。

## 四层范式

| 范式 | 抽象层级 | 解决什么 | 一句话定义 |
|------|---------|---------|-----------|
| **Prompt Engineering** | L1 | 怎么表达任务 | 单次任务表达与输出约束 |
| **Context Engineering** | L2 | 给什么信息 | 模型可见的信息环境 |
| **Harness Engineering** | L3 | 怎么约束和验证 | Agent 的工具、权限、环境与检查器 |
| **Loop Engineering** | L4 | 怎么迭代闭环 | 目标 → 计划 → 执行 → 验证 → 修正的循环 |

## 层级关系

```
┌─────────────────────────────────────┐
│ L4 Loop Engineering                 │ ← 设计目标驱动的迭代闭环
│  ┌─────────────────────────────┐   │
│  │ L3 Harness Engineering       │   │ ← 设计 Agent 的运行环境
│  │  ┌─────────────────────┐   │   │
│  │  │ L2 Context Eng.      │   │   │ ← 设计 Agent 可见的信息
│  │  │  ┌─────────────┐   │   │   │
│  │  │  │ L1 Prompt    │   │   │   │ ← 单次任务的表达
│  │  │  │ Engineering  │   │   │   │
│  │  │  └─────────────┘   │   │   │
│  │  └─────────────────────┘   │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

外层**包裹**内层,不是**替代**内层。Loop 内部每一轮迭代仍然需要 Prompt / Context / Harness 三者配合。

## 何时使用哪一层

| 任务特征 | 用哪一层 |
|---------|---------|
| 一次性小任务、解释、翻译 | L1 Prompt |
| 多文件协同、风格一致性、大型项目重构 | L2 Context |
| 高可靠模块、CI/CD、安全敏感变更 | L3 Harness |
| 高度重复 + 强验证器 + 风险可控 + 预算充足 | L4 Loop |

## 时间线

| 时间 | 范式 | 标志 |
|------|------|------|
| 2022-2024 | Prompt Engineering | ChatGPT、Copilot 主导 |
| 2025 起 | Context Engineering | 系统化讨论,RAG、记忆系统、Wiki 兴起 |
| 2025 下-2026 | Harness Engineering | Claude Code / Codex / Cursor 成为主流 |
| 2026 中 | Loop Engineering | Boris Cherny / Peter Steinberger / Addy Osmani 提出 |
