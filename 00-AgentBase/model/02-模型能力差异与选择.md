# 模型能力差异与选择

> 本文讲解 Agent 中"模型"的差异、选型考量、版本升级影响。
>
> 概念辨析见根级 [`../02-概念全景辨析.md`](../02-概念全景辨析.md)。

## 1. 模型在 Agent 中的角色

**模型 = Agent 的大脑**，但**不是全部**。

Agent 的完整组成：

```
┌─────────────────┐
│   模型 (LLM)    │ ← 大脑:推理、决策、生成
└────────┬────────┘
         ↓
┌─────────────────┐
│   上下文(状态)   │ ← 记忆:对话历史、工具结果、记忆
└────────┬────────┘
         ↓
┌─────────────────┐
│   工具 / 技能    │ ← 手脚:操作世界
└────────┬────────┘
         ↓
┌─────────────────┐
│   Runtime       │ ← 操作系统:权限、沙箱、调度
└─────────────────┘
```

**模型只负责**：接收输入（提示+上下文）→ 推理 → 输出（文本/工具调用）。其他都不在模型的职责范围内。

### 2.2 选择决策树

```
需要最强推理 + 长上下文?
├─ 是 → Claude Opus 4(首选) 或 GPT-5(备选)
└─ 否 →

需要专攻代码生成?
├─ 是 → Codex(gpt-5-codex) 或 Claude Sonnet 4
└─ 否 →

需要超长上下文(>500K)?
├─ 是 → Gemini 2.5 Pro(1M 上下文)
└─ 否 →

需要便宜?
├─ 是 → Claude Haiku 或 DeepSeek V3
└─ 否 → 看具体场景
```

## 3. 模型升级的影响

升级模型版本可能带来：

| 维度 | 影响 |
|------|------|
| 工具调用格式 | 可能改变(检查 schema 兼容性) |
| 长上下文窗口 | 可能扩大 |
| 推理风格 | 可能改变(Prompt 调优需重做) |
| 拒绝行为 | 安全策略变化(可能更严或更宽松) |
| 价格 | 升降变化 |
| 输出格式 | 微调建议/代码风格变化 |

**升级前必做**:
- [ ] 在测试集跑一遍
- [ ] 检查 trace 中的工具调用格式
- [ ] 评估 Prompt 是否仍适用
- [ ] 验证记忆/上下文兼容

## 4. 模型 ≠ Agent

**常见误解**:换模型 = 换 Agent。

实际上:
- **同一个模型** + 不同 Prompt / 工具 / 上下文 → 行为差异巨大
- **不同模型** + 同一套环境 → 行为仍有差异,但适配良好的环境差异可控
- **Agent 是系统**,模型只是其中一个组件
