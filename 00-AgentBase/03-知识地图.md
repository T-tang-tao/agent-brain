# Agent 知识地图

## 一句话解释

Agent 知识地图用来把 Agent 体系中的 Model、Runtime、Tools、Knowledge、Behavior、Safety、Deployment 连接起来，帮助人和 Agent 快速理解整个系统。

> 人类入门内容(基础概念解释、类比、误解澄清)在 [00-human/](./00-human/README.md)，Agent 不需要读。本文件面向已具备基础认知的读者和 Agent。

## 这张图怎么读

### Model

Model 是 Agent 的推理和生成能力。

它不是 Agent 的全部，只是 Agent 的核心能力来源。

### Runtime

Runtime 是 Agent 运行的环境。

Claude Code CLI、Codex CLI、Hermes Agent 这类系统都可以理解为不同形式的 Agent Runtime。

> **本知识库的 Runtime 模块视角**:`runtime/` 只讲 Runtime 的**安装、部署和使用**(运维视角)。Runtime 提供的机制能力(可观测、记忆、规划循环等)作为 Agent 的"行为机制",放在 `behavior/` 下。

> **当前使用的 Runtime**:Trae(独立 exe,项目级目录约定 `.trae/`),只在 [runtime/00-项目级配置](./runtime/00-项目级配置.md) 中提及,不单独建专题。

### Tools

Tools 是 Agent 能调用的动作。

工具可以是内置工具，也可以通过 MCP 暴露，也可以是自定义工具。

### Knowledge

Knowledge 是 Agent 可以读取或检索的信息。

包括 wiki、项目文档、RAG、代码文件等。

> **Memory 的归属**:Memory(短期/长期/实体/知识图谱)是 Agent 的"行为能力"而非"知识库设计方法",归到 [behavior/04-记忆机制(Memory)](./behavior/04-记忆机制(Memory).md)。

### Behavior

Behavior 是 Agent 行为方式的来源。

包括 Prompt、Skill、Planner(规划循环)、Memory(记忆机制)、Evaluation(评估方法)、Observability(可观测性)。

- **Prompt**:当前怎么行动
- **Skill**:某类任务怎么做
- **Planner**:怎么规划-执行-反思([behavior/03](./behavior/03-规划循环(Planner).md))
- **Memory**:怎么记住和检索([behavior/04](./behavior/04-记忆机制(Memory).md))
- **Evaluation**:怎么衡量表现([behavior/05](./behavior/05-评估方法(Evaluation).md))
- **Observability**:怎么审计行为([behavior/06](./behavior/06-可观测性(Observability).md))
- **Context 工程**:怎么为单次推理装高信号 context([behavior/09](./behavior/09-Context-Engineering.md))
- **Filesystem Context**:怎么用文件系统扩展 context([behavior/10](./behavior/10-Filesystem-Context.md))
- **Multi-Agent 模式**:怎么拆 Agent、supervisor/peer/hierarchical([behavior/11](./behavior/11-Multi-Agent-Patterns.md))
- **Tool 设计**:工具契约、合并、命名空间([behavior/12](./behavior/12-Tool-Design.md))
- **Harness 工程**:模型之外的控制层、surface 分类、autonomy([behavior/13](./behavior/13-Harness-Engineering.md))
- **LLM 项目开发**:task-model fit + staged pipeline([behavior/14](./behavior/14-LLM-Project-Development.md))
- **Hosted Agents**:后台沙箱、多人协作、warm pool([behavior/15](./behavior/15-Hosted-Agents.md))
- **Agent 架构模式**:ReAct / Plan-Execute / Reflexion([behavior/08](./behavior/08-Agent架构模式.md))
- **开发型 Agent 工作协议**:读上下文 → 最小改动 → 验证([behavior/07](./behavior/07-开发型Agent工作协议.md))

### Safety

Safety 是 Agent 的边界和验证系统。

越能行动的 Agent，越需要明确限制和验证。Safety 模块覆盖三个方面：Permission（权限控制）、Boundary（行为边界）、Verification（完成验证）。

> **关于 Evaluation 和 Observability**:之前归在 Safety 下,现在归到 Behavior 下——它们本质是 Agent 行为质量的度量和审计手段,不是"安全"。

### Deployment

Deployment 是 Agent 的部署模式与生命周期管理。

包括本地交互式、服务端长驻、嵌入式集成三种典型模式,以及环境隔离、CI/CD、密钥管理、版本管理等运维主题。详见 [deployment/01-部署模式总览](./deployment/01-部署模式总览.md)。

## 概念之间的关系

```text
Prompt 告诉 Agent 当前怎么行动。
Skill 告诉 Agent 某类任务怎么做。
Tool 让 Agent 能执行动作。
MCP 让 Agent 能连接外部工具和资源。
Knowledge Base 提供 Agent 可以查的资料。
Memory 提供 Agent 长期保存的经验。
Planner 决定 Agent 怎么拆解和执行任务。
Evaluation 衡量 Agent 表现好不好。
Observability 让 Agent 行为可审计。
Boundary 限制 Agent 什么不能做。
Verification 检查 Agent 是否真的完成。
Deployment 决定 Agent 怎么跑起来。
```

## 迁移时的使用方式

未来迁移 Hermes Agent、Claude Code CLI Agent、Codex Agent 时，可以按这张图检查：

1. 新环境使用什么 Model？
2. Runtime 怎么安装、部署？
3. Tools 是否等价？
4. MCP 是否可接入？
5. Skill 是否能迁移？
6. Prompt 是否需要改写？
7. Knowledge Base 是否能读取？
8. Memory 如何迁移？
9. Planner 模式是否兼容？
10. Boundary 是否仍然有效？
11. Verification 如何执行？
12. 部署模式怎么调整？
