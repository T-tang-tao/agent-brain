# Memory Write Policy

> 记忆写入边界。Agent 写到长期记忆 / 上下文 / RAG 索引前必须参考本策略。

## 规则名称

`memory-write-policy`

## 适用范围

- 所有 agent
- 所有记忆层写入:长期记忆 / 短期记忆 / RAG 索引 / vector store / 知识库正文
- 任何记忆系统(Claude Code Memory / Hermes Memory / 自建向量库)

## 允许的行为

✅ Agent 可自主执行:

- **写学习到的项目约定**:命名规范、目录结构、本仓库习惯
- **写用户偏好**:语言(中/英)、输出格式偏好、风格偏好
- **写已知的事实**:API 行为、库版本、配置项
- **写"忘记"清单**:用户明确说"不要记住 X",必须能删

## 需要确认的行为

⚠️ 必须先问用户:

- **写敏感 PII**:身份证、银行卡、手机号、地址
- **写密码 / token / API key**:即使加密也不行
- **写用户健康 / 财务 / 关系信息**:敏感类别
- **写"未来行为假设"**:用户说"以后都用 X",需确认是否真的要持久化
- **写跨会话的猜测**:agent 推断而非用户明确说的

## 禁止的行为

❌ 任何情况下禁止:

- **把 token / 密码写进任何记忆**:即使是 base64 编码也不行
- **把实时数据写进长期记忆**:价格、订单、账户余额(用工具查)
- **把"幻觉"写进记忆**:agent 编造的、没验证的信息
- **把临时调试输出写进记忆**:日志、堆栈、临时变量
- **写入未压缩的全量上下文**:占太多空间
- **写入第三方 SDK / 库的内部状态**:易变、容易过期
- **写入后再读取循环引用**:记忆 A 引用 B,B 引用 A
- **永久写入敏感可识别信息**:哪怕用户授权一次,也不应永久

## 风险说明

| 风险 | 后果 |
|------|------|
| 密钥进记忆 | 任何读取记忆的人都能看到 |
| PII 永久化 | GDPR / CCPA 合规问题 |
| 实时数据进知识库 | 决策基于过期数据 |
| 幻觉传播 | 后续对话引用错的"事实" |
| 上下文爆炸 | 单次会话记忆过多,模型困惑 |
| 跨用户泄露 | 多租户场景下错把 A 的记忆给 B |

## Agent 执行说明

1. **判断内容**:
   - 是事实/约定/偏好? → [允许]
   - 是敏感/PII/密钥? → [禁止]
   - 是用户明确说"记住这个"? → [允许]
   - 是 agent 推断? → [需确认]
2. **判断时长**:
   - 单次会话有用? → 短期记忆即可
   - 跨会话需要? → 长期记忆 + 用户确认
3. **判断可删除性**:
   - 用户说"忘掉这个"必须能删 → 用支持 delete 的存储
4. **写入后**:
   - 简短报告"已记住 X"
   - 给出查看/删除方式

## 例子

### 允许

```text
> 记住:用户偏好用中文回复,代码注释也用中文
```

✅ 写到长期记忆

### 需确认

```text
> 我注意到你每次都用 max_position = 0.05,要不要记住?
```

⚠️ "这是交易参数,不是普通偏好。要确认:1) 你真的想持久化 2) 写进哪里(知识库还是 .env) 3) 怎么回滚?"

### 禁止

```text
> 把我的 OKX API key 存到长期记忆里,以后自动用
```

❌ "禁止。密钥永远不进记忆。用环境变量 OKX_API_KEY,文档里写引用方式不写值。"

## 记忆系统对比

| Runtime | 短期 | 长期 | 删除 |
|---------|------|------|------|
| Claude Code | 会话内 context | CLAUDE.md / 项目级 memory | 改文件 / 清会话 |
| Hermes Agent | ephemeral default | file:// 可配置 | 删文件 |
| Codex CLI | 会话内 | AGENTS.md / .codex/ | 改文件 |
