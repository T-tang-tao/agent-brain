# 边界规则模板

## 规则名称

写清楚这条边界规则的名称。

## 适用范围

说明规则适用于哪些 agent、哪些目录、哪些任务或哪些工具。

例如：

- 适用于所有 coding agent；
- 适用于 shell 命令；
- 适用于外部 API；
- 适用于交易相关任务。

## 允许的行为

Agent 可以自主执行的行为：

- ...
- ...

## 需要确认的行为

Agent 必须先询问用户或协调者的行为：

- ...
- ...

## 禁止的行为

Agent 不允许执行的行为：

- ...
- ...

## 风险说明

解释为什么这些行为需要限制。

例如：

- 不可逆；
- 可能删除数据；
- 可能产生费用；
- 可能影响生产环境；
- 可能泄露敏感信息；
- 可能触发真实交易或外部副作用。

## Agent 执行说明

如果 Agent 读取本规则，应该如何应用：

1. 先判断当前任务是否属于适用范围。
2. 如果行为在允许列表中，可以执行。
3. 如果行为在需要确认列表中，必须先询问。
4. 如果行为在禁止列表中，必须拒绝或改用安全替代方案。
5. 执行后必须汇报结果和验证情况。

## 例子

### 允许

- 示例：读取项目文件。

### 需要确认

- 示例：删除目录。

### 禁止

- 示例：把 API key 写入长期记忆。
