# AGENTS.md 定制指南

> 本文件供 kb-init 技能在 Step 4(生成 AGENTS.md)时参考。说明不同类型知识库的 AGENTS.md 应如何定制。

## 10 个必需章节

无论什么类型,AGENTS.md 都必须包含:

1. **知识库定位**:一句话——这是什么知识库、给谁用、解决什么问题
2. **目录结构**:完整目录树 + 每个目录的职责说明
3. **写作标准**:人话优先、类比驱动、可视化、可复述
4. **文件命名规范**:编号规则、命名约定
5. **元数据标准**:每个文档必须包含的元数据表格
6. **防信息孤岛规则**:入口≥1、出口≥1、零死链、双向引用
7. **知识编译流程**:从原始材料到知识库的编译步骤
8. **入库工作流**:发现缺口 → 起草 → 自检 → 审阅 → 发布 → 更新索引
9. **审查清单**:发布前必须通过的检查项
10. **版本管理**:语义化版本规则

## 按类型定制

通用章节保持一致,以下章节需按知识库类型强化:

### 编程/项目知识库

- **写作标准**强化:代码示例必须标注语言、版本;ADR 要记录决策背景、考虑的替代方案、选择理由
- **审查清单**强化:代码示例是否可运行?ADR 是否完整?API 文档是否与代码同步?
- **编译流程**强化:从 issue/PR 摘要 → 提炼为架构决策或排障文档

### 产品开发知识库

- **写作标准**强化:每个需求文档(PRD)必须包含"背景→目标→验收标准→非目标"四要素;功能 spec 要记录状态(草案/已确认/已实现/已废弃);决策记录要标注是产品决策还是技术决策
- **审查清单**强化:需求是否标注了验收标准?功能 spec 的状态是否与代码实际一致?实时进度(Sprint/burndown/bug 数)是否被误写入库?里程碑是否有过期标注?
- **编译流程**强化:从需求评审会议记录 → 提炼为 PRD;从技术讨论 → 提炼为 ADR;从上线后总结 → 提炼为复盘文档
- **额外强调**:实时项目进度(Sprint 状态、bug 列表、CI 结果、代码提交)禁止写入知识库,必须通过工具(Jira/GitHub Issues/CI)查询;知识库只保留**静态决策与经验**(为什么这样设计、需求怎么来的、上次踩了什么坑)

### 交易/量化知识库

- **写作标准**强化:必须明确标注"静态知识"vs"实时状态";策略说明要包含参数定义、适用条件、失效条件
- **审查清单**强化:是否有实时数据被误写入库?策略是否标注了失效条件?
- **编译流程**强化:从回测记录 → 提炼为策略经验;从交易日志 → 提炼为风控规则
- **额外强调**:实时状态(价格、订单、账户余额)禁止写入知识库,必须通过工具/API 查询

### 学术/研究知识库

- **写作标准**强化:每个论断必须标注来源;矛盾信息以权威来源(同行评审论文)仲裁
- **审查清单**强化:引用是否可溯源?概念定义是否消歧?
- **编译流程**强化:从论文阅读笔记 → 提炼为概念定义或文献综述

### 客服/文档知识库

- **写作标准**强化:每个回答必须引用权威来源;禁止编造;版本变更要标注生效日期
- **审查清单**强化:回答是否引用了权威来源?是否有版本标记?过期内容是否已标记?
- **编译流程**强化:从客服对话记录 → 提炼为 FAQ;从政策变更 → 更新 policies

### 个人助手知识库

- **写作标准**强化:标注隐私等级(公开/个人/敏感);敏感信息不写入
- **审查清单**强化:是否有敏感信息泄露?Memory 是否小而准?
- **编译流程**强化:从日常对话 → 提炼为偏好或习惯记录
- **额外强调**:可删除、可纠正,用户有权随时清除特定记忆

## 00-human 人类入门区规则

如果知识库包含 `00-human/` 目录,AGENTS.md 中需补充以下规则:

1. **目录结构**中标注 `00-human/` 为"人类入门(Agent 不需要读)"
2. **写作标准**增加一条:00-human 文档风格为叙述式、讲故事、类比丰富,可超过 300 行;其他模块保持结构化、查表式
3. **禁止事项**增加例外:超过 300 行限制对 `00-human/` 豁免
4. **编号规则**增加:00-human 从 01 起独立编号
5. **入库工作流**增加:判断内容属于人类入门还是项目特定——如果是"教人类理解概念"则放 00-human,如果是"记录项目规则/约定"则放对应模块

判断标准:

| 内容类型 | 放哪里 | 例子 |
|----------|--------|------|
| 教人类理解概念 | `00-human/` | "什么是 Agent""Tool 和 Skill 有什么区别" |
| 记录项目结构 | 对应模块 | "这个知识库的目录怎么组织" |
| 记录项目规则 | 对应模块 | "哪些操作需要确认""命名规范是什么" |
| 记录项目约定 | 对应模块 | "代码示例必须标注语言版本" |
