# Interview — AI 引导式问卷设计

> 本文件定义 `project-init` Step 2 的问卷结构、问题话术、跳过策略。

## 0. 核心原则

1. **可一句给完**:用户可能一句话讲完所有信息,不要把已经知道的拆成 4 个问题再问
2. **可一键跳过**:用户答"通用 + 待定"就走默认值
3. **追问而不是预判**:前 3 个问题必答,后续追问按回答展开
4. **预算 ≤ 6 轮对话**:超过 6 轮说明项目太复杂,拆成多轮初始化

## 1. 四个核心问题

### Q1:项目做什么

**默认话术**:
> 这个项目做什么?一句话 + 给我一个标签(可选:web / backend / 数据 / 内容 / agent / 通用)

**期望回答形式**:
- 一句话:这个项目是一个面向 X 的 Y 工具
- 标签:web | backend | 数据 | 内容 | agent | 通用

**示例**:
- "内部使用的 Markdown 转 PDF 工具,标签内容"
- "公司官网 + 博客,标签 web"
- "AI 客服后端,标签 agent"

**追问**(按需):
- 用户群是谁?
- 解决什么痛点?
- 竞品 / 替代品是什么?

### Q2:技术栈

**默认话术**:
> 主要语言 / 框架 / 部署目标?不知道就写"待定"

**期望回答形式**(逗号分隔即可):
- `Python, FastAPI, PostgreSQL, Docker`
- `TypeScript, Next.js, Vercel`
- `待定`

**追问**(按需):
- 包管理器(pip / npm / pnpm / bun)?
- 测试框架?
- CI/CD 平台?

### Q3:团队规模 / 阶段

**默认话术**:
> 团队多大?项目在什么阶段?(mvp / 增长 / 稳定)

**期望回答形式**:
- `1 人 + mvp`
- `3 人 + 增长`
- `10 人 + 稳定`

**为什么问这个**:
- `mvp`:推荐轻量 skill(`development-agent` + `superpowers` 核心),不要企业级
- `增长`:推荐 `superpowers` 全套 + 测试覆盖
- `稳定`:推荐 `superpowers` + 严格边界规则

### Q4:边界要求

**默认话术**:
> 有什么安全 / 边界要求?(密钥、提交限制、必须确认的操作、禁区)
> 没就写"无"或"待定"

**期望回答形式**:
- `生产密钥不入仓 / 提交前必须跑测试 / 不要碰 migrations/`
- `无`
- `待定`

**追问**(按需):
- 是否限制网络访问?
- 是否限制某些工具(如 `webfetch`)?
- 是否需要审批流程?

## 2. 跳过策略

### 2.1 一句话模式

如果用户开场给的信息已经覆盖大部分:

> "做个 AI 客服后端,Python+FastAPI,1 人 mvp,生产密钥不入仓"

直接进 Step 3,不需要追问。

### 2.2 一次性问答

每个问题用户回答完立即追问,不要等所有问题问完再总结。

### 2.3 完全跳过

用户答"全部默认"或"按推荐走",直接:
- 标签 = `通用`
- 技术栈 = `待定`
- 阶段 = `mvp`
- 边界 = `无`

然后走默认 skill 组合:`superpowers` + `agents-md-author`。

## 3. 反模式

不要问这些(用户不愿意答):
- ❌ "商业模式是什么?"(隐私 + 太宽泛)
- ❌ "未来 3 年规划?"(太虚)
- ❌ "请列出所有依赖版本"(细节,Step 5 部署时再说)

## 4. 输出格式

问卷结束后,合并成结构化对象:

```yaml
project:
  purpose: 一句话项目描述
  tag: web | backend | 数据 | 内容 | agent | 通用
  stack:
    language: Python
    framework: FastAPI
    database: PostgreSQL
    deploy: Docker
    package_manager: uv
  team:
    size: 1
    stage: mvp
  boundaries:
    - 生产密钥不入仓
    - 提交前必须跑测试
    - 不要碰 migrations/
```

这个对象供 Step 3 匹配、Step 4 生成 AGENTS.md 使用。
