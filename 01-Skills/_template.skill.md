# Skill 名称

## 用途

说明这个 skill 用来解决什么问题。

## 触发条件

什么时候应该使用这个 skill。

例如：

- 用户请求调试 bug；
- 用户请求 code review；
- 用户请求迁移 agent；
- 用户请求整理 prompt。

## 不适用场景

什么时候不应该使用这个 skill。

例如：

- 任务过于简单；
- 不需要该流程；
- 缺少必要上下文；
- 用户明确要求不要使用。

## 输入

使用这个 skill 前需要哪些上下文：

- 用户目标；
- 项目文件；
- 相关日志；
- 现有 prompt；
- 运行环境；
- 限制条件。

## 执行流程

1. 明确任务目标。
2. 收集必要上下文。
3. 按步骤执行。
4. 在关键节点验证。
5. 输出结果和剩余问题。

## 检查清单

- [ ] 是否理解用户目标？
- [ ] 是否确认适用范围？
- [ ] 是否收集必要上下文？
- [ ] 是否遵守边界规则？
- [ ] 是否完成验证？
- [ ] 是否如实报告失败或跳过的步骤？

## 边界

说明这个 skill 中哪些行为：

- 可以自主执行；
- 需要用户确认；
- 禁止执行。

可参考：

- [[../00-AgentBase/safety/01-Agent边界与限制]]
- [[../05-Boundaries/_boundary-policy-template]]

## 输出

说明使用这个 skill 后应该产出什么。

例如：

- 修改后的文件；
- review findings；
- 迁移清单；
- prompt 草案；
- 验证结果。
