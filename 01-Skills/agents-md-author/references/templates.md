# AGENTS.md 模板

## 完整模板(用于完全重写)

```markdown
# AGENTS.md — <目录名> 管理规范

> <一句话说明本目录的职责>
>
> <指向根 AGENTS.md 或相关总览>

---

## 1. 核心理念

1. <铁律 1>
2. <铁律 2>
3. <铁律 3>

---

## 2. 目录结构

```
<目录名>/
├── AGENTS.md                  ← 本文件
├── <子项 1>
│   └── ...
└── <子项 2>
```

---

## 3. 管理规范

| 操作 | 流程 |
|------|------|
| <操作 1> | <流程> |
| <操作 2> | <流程> |

---

## 4. 边界

✅ 允许:<可自主执行>
⚠️ 需确认:<需用户确认>
❌ 禁止:<任何情况禁止>

---

## 5. 自检清单

- [ ] <项 1>
- [ ] <项 2>

---

## 相关

- <相关文档 1>
- <相关文档 2>
```

## 常见反模式

### 反模式 1:把 AGENTS.md 当 README 写

```markdown
❌ # 我的项目

这是一个很棒的项目,它始于 2024 年,由 ... 团队开发,致力于 ...
```

```markdown
✅ # AGENTS.md — 项目目录管理规范

> 本目录是 <项目> 的代码根。Agent 在此目录下的行为约束如下。
```

### 反模式 2:解释每个决定的"为什么"

```markdown
❌ ## 为什么用 SQLite 而不是 PostgreSQL?
[300 字解释]
```

```markdown
✅ ## 数据存储
用 SQLite。详见 [ADR-001](docs/adr/001-sqlite.md)。
```

### 反模式 3:用"我们认为"开头

```markdown
❌ 我们认为,也许可以考虑,在某些场景下,文件可能需要先备份...
```

```markdown
✅ 删除文件前必须先备份。
```
