# AGENTS.md 风格规则

> 本文件是 [`../../../00-AgentBase/knowledge/04-如何编写AGENTS.md`](../../../00-AgentBase/knowledge/04-如何编写AGENTS.md) 的精简版,作为 Skill 触发后的快速检查清单。

## 1. 检查清单(写完必过)

- [ ] 没有"为什么"段落
- [ ] 没有重复 00-AgentBase/ 已有内容
- [ ] 没有重复 README.md 已有内容
- [ ] 规则用祈使句
- [ ] 表格用结构化数据
- [ ] 边界规则覆盖三类(✅ ⚠️ ❌)
- [ ] 没有模糊词(我们、可能、也许)
- [ ] 引用而非重复
- [ ] 长度 < 500 行

## 2. 必须用的写法

### 祈使句

```markdown
✅ 允许:读所有文件
⚠️ 需确认:写 00-AgentBase/
❌ 禁止:覆盖 .obsidian/
```

### 结构化表格

```markdown
| 字段 | 值 |
|------|-----|
| name | my-skill |
| version | 1.0.0 |
```

### 引用而非重复

```markdown
见 [00-AgentBase/behavior/01-如何编写Skill.md](...)
```

## 3. 必须避免的写法

### 自问自答

```markdown
❌ "## 为什么这样设计?"
❌ "## 设计动机"
❌ "## 与其它方案的对比"
```

### 模糊词

```markdown
❌ "我们认为..."
❌ "也许可以考虑..."
❌ "在某些场景下..."
❌ "建议..."
```

### 散文式规则

```markdown
❌ "当一个东西刚被创建还没验证的时候就叫验证中..."
✅ | 验证中 | 刚创建,未验证 |
```

### 重复知识库

```markdown
❌ [在 AGENTS.md 里写 200 行解释 Skill]
✅ 见 00-AgentBase/behavior/01-如何编写Skill.md
```

## 4. AGENTS.md 应当明确

| 内容 | 示例 |
|------|------|
| 核心理念 | 3-5 条铁律,每条 1 行 |
| 目录结构 | ASCII tree |
| 管理规范 | 入库/更新/删除/审计 |
| 边界 | 允许/确认/禁止 |
| 自检 | 操作前过一遍 |

## 5. 不应当出现的章节

```
为什么 / Why
动机 / Motivation
设计取舍 / Trade-offs
历史背景 / Background
类似项目 / Similar Projects
完整原理 / Theory
未来改进 / Future
```

**这些内容放到**:
- `99-Roadmap.md` 决策日志(决策)
- `00-AgentBase/` 知识库(原理)
- 项目 `README.md`(背景与类似项目)
- 新文档(完整原理)

## 6. 长度阈值

| 范围 | 建议 |
|------|------|
| < 200 行 | 正常 |
| 200-500 行 | 检查是否在写教科书 |
| > 500 行 | 必须拆分,或删除冗余 |
